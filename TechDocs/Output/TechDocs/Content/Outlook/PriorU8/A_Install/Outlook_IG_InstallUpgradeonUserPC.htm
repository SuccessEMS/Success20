Installing/Upgrading EMS for Outlook on a User's PC

Note: If you are installing on a machine with Outlook 64bit, use the 64bit (x64) EMS for Outlook installer.  If you are installing on a machine with Outlook 32bit, use the 32bit (x86) EMS for Outlook installer. 64-bit editions of Microsoft Office 2010 and 2013 require additional installation steps not outlined in this document.  Please contact Customer Support for more information.  

 

Note: The Update 7 release requires that users run Microsoft® Outlook in cached mode.

Silent EMS for Outlook User Installation 

You can push your EMS for Outlook Installation to user machines if your system enables this type of administration.

Use the following command to establish an Unattended/Silent installation of the EMSForOutlook.msi (replace [ServerName] with the name of your web server):

msiexec /I “EMS for Outlook.msi” /quiet ENDPOINT_ADDRESS=”http://[ServerName]/ExchangeIntegrationWebService/Service.asmx”

Standard EMS for Outlook User Installation 

  1. Verify that the pre-installation requirements have been met.
  2. Download the EMSForOutlook.msi file onto the user’s desktop.
  3. Close Outlook.
  4. Run EMSForOutlook.msi.
  5. The first screen welcomes you to the EMS Outlook Add-in Setup Wizard.  Click the Next button to begin the installation process.  The Destination Folder screen will appear.

  1. Specify the installation Folder.  Note - It is recommended that you keep the default setting. 
  2. Click the Next button. The Outlook Integration Service Information screen will appear.
  3. Enter the Outlook Integration Web Service address your organization uses. (Example - http://[ServerName]/ExchangeIntegrationWebService/Service.asmx)
  4. Click the Next button. The Ready to install EMS for Outlook screen will appear. 
  5. Click the Install button to complete the installation.  Click the Close button to exit.
  6. Launch Outlook. The EMS button should display on the user’s Outlook toolbar.

Warning: EMS for Outlook requires Outlook to be running in cached mode (the option selected under File > Account Settings) as shown below). The add-in has a close tie to Outlook and makes frequent calls as users click through the Outlook interface. In cached mode, over 95% of those calls end at the local Outlook cache file. If cached mode is disabled, however, those calls end at Exchange and the frequency of calls can cause performance issues. For this reason, the add-in will become inactive if Outlook is not in cached mode, and users will receive a message to this effect.
