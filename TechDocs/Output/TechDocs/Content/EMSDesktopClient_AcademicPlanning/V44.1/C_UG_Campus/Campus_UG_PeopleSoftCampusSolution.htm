PeopleSoft Campus Solution Integration

A stored procedure into EMS_SPUPDATECOURSEDATES provides additional updates to Peoplesoft.

To perform a full location writeback for PeopleSoft Campus Solutions:

  1. Update the PS_CLASS_MTG_PAT record with the room ID.
    1. If an Event ID already exists, update the existing PS_CAMPUS_MTG records.
    2. If an Event ID does not exist:
      1. Update the PS_EVENT_LAST_TBL table – increment by one.
      2. Retrieve the EVENT_NBR_LAST value from the PS_EVENT_LAST_TBL table.
      3. Update the PS_CLASS_TBL record with the Event ID.
      4. If it’s a combined section, update the related combined sections in PS_CLASS_TBL with the Event ID, so long as the SKIP_MTGPAT_EDIT flag is set to ‘No.’
  2. Create PS_CAMPUS_EVENT record if it doesn’t exist for the Event ID.
  3. Create PS_CAMPUS_MTG records if they don’t exist for the Event ID and Meeting Number.

Note: The Oracle account used for the integration with EMS will need to be updated to have write access to the following tables, if it does not already:
PS_CLASS_MTG_PAT
PS_EVENT_LAST_TBL
PS_CLASS_TBL
PS_CAMPUS_EVENT
PS_CAMPUS_MTG
