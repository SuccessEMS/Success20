Fixes  

Area

Description

Log In

Keyboard Covered Code Field on Login Screen When 2FA is Enabled

When users attempted to log in with 2FA enabled, the verification Code field was blocked by the keyboard.  

Fix: Modified the display so the user can see the verification Code field in this situation.

Internal Reference # 27741
(For EMS Internal Use Only)

Scan QR Code

"Reservation Created" Toast Absent When Using a Second QR Code

When users booked a reservation by scanning a QR code for the second time in a session, the confirmation toast "Reservation Created" did not appear.

Fix: Modified the display to show the toast message now appears when a QR code is used a second time.

Internal Reference # 27744
(For EMS Internal Use Only)

Meeting Prematurely Listed As In Progress

When users scanned a QR code to see events for a room, some events showed as In Progress up to two hours prior to start time.

Fix: Modified time zone offset calculations so that rooms only show as In Progress upon start time.

Internal Reference # 28519
(For EMS Internal Use Only)

Auto-Create Workflow Issues

When users scanned a QR code to automatically create a booking, several issues arose. Where there was only one process template, they still had to click the "+" button to initiate the booking process. In a fully automated scenario where all field values were defaulted, they still had to click Create Reservation to finalize the booking.

Fix: Modified the automation workflow so users are take directly to the booking process when they must complete some fields, and so that bookings are automatically finalized in fully defaulted scenarios.

Internal Reference # 28694 & 27974
(For EMS Internal Use Only)

Selecting a Process Template After Scanning QR Code Returned Users to Scan QR Code Screen

When users scanned a QR code to book a room and clicked on a certain process template from the available list, they were incorrectly returned to the Scan QR Code page.

Fix: Modified validations so users can select the process template and proceed with the booking as expected.

Internal Reference # 27966
(For EMS Internal Use Only)

iOS: Disabling Camera Caused Crash When Scanning  a QR Code

When users scanned a QR code using iOS devices that had the camera disabled for the first time, the app crashed.

Fix: Modified validations to successfully load the app when Allow Camera is off.

Internal Reference # 27947
(For EMS Internal Use Only)

Create Reservation

Geolocation Issues

When users booked a room and Location Services was turned on, the Distance field was empty and the list of room results was not in order of proximity. On Android devices, when Location Services was turned off, the Distance label persisted and kilometers displayed as "NaN Kilometers;" additionally, the default building/view for the booking template in use was not automatically selected. 

Fix: Modified functionality so that when Location Services is turned on, users can see the distance of a room and the results list is sorted by nearest to furthest, and for Android, when Location Services is turned off, users no longer see the distance, and kilometers and default buildings/views display as expected.

Internal Reference # 28406 & 28596
(For EMS Internal Use Only)

Tapping Blue Arrow Caused Crash

When users tapped the blue FormLink "Next" arrow on the right from booking details or new reservation screens, the app crashed.

Fix: Modified functionality so that tapping the arrow no longer causes a crash.

Internal Reference # 28498
(For EMS Internal Use Only)

Start and End Times Defaulting Incorrectly

When users booked a meeting too close to closing time (according to booking rules), start and end times defaulted to the same time. Additionally, when booking rules set default meeting duration to 45 minutes after start time, default end time was calculated incorrectly.

Fix: Modified functionality so that template defaults are properly applied and default end times are correctly calculated.

Internal Reference # 28374 & 28332
(For EMS Internal Use Only)

Changing Location Turned Off Capacity Filter For Room Search

When users searched for rooms and filtered by room capacity, and then changed location, the capacity filter turned off.

Fix: Modified functionality so that changing the location does not turn off the capacity filter.

Internal Reference # 28507
(For EMS Internal Use Only)

Child UDF Default Values Missing

When a booking template was set to show the default values set for child UDFs and the parent UDFs allowed multiple selections, the default values of the child fields did not display. 

Fix: Modified the display to show child UDF default values of multi-select parent UDFs in this situation.

Internal Reference # 27920
(For EMS Internal Use Only)

Edit Reservation

"Room Unavailable" Error When Changing Event Time To Follow Another Event

When users edited an event so that it started immediately after another one in the same room, the error "Room Unavailable" presented incorrectly and their room selection was cleared. 

Fix: Modified validations to allow adjacent scheduling when a meeting has no setup and teardown allowance.

Internal Reference # 28458
(For EMS Internal Use Only)

Unable to Add Attendees After Booking Exchange Meeting

When users added attendees to an Exchange-compatible meeting that had not been originally created with attendees, their additions were not saved and attendees were not invited. 

Fix: Modified functionality so that users can successfully add attendees when editing in this situation.

Internal Reference # 28456
(For EMS Internal Use Only)

Rooms Not Listed Under View

When users edited an event and tapped the View option for a selected room, it was not in the list as expected.

Fix: Modified the display so the selected room shows in both Building and View as expected.

Internal Reference # 28560
(For EMS Internal Use Only)

Favorite Room Listed Twice

When users edited an event to move a meeting later on the schedule, the selected favorite room was listed twice.  

Fix: Modified the display so the selected favorite room only lists once as expected.

Internal Reference # 28401
(For EMS Internal Use Only)

Time Zones

Time Zones Without Designated Abbreviations Displayed as "NULL"

Time zones that did not have abbreviations defined in the user database showed as "NULL" on the app.  

Fix: Modified functionality so that when time zone abbreviations are not defined, the time zone field shows the full name of the time zone.

Internal Reference # 28367
(For EMS Internal Use Only)

Incorrect Time Zone Adjustment When Changing Event Date

When users changed an event date such that it was after the Daylight Savings Time period for their devices, the event time was incorrectly adjusted.  

Fix: Modified functionality so that time zone adjustments are made correctly in this situation.

Internal Reference # 28421
(For EMS Internal Use Only)

Room Incorrectly Unavailable When Server Time Zone Was "UTC +00"

When the server time zone was set to UTC + 00 and users picked a room spanning Daylight Savings Time, the room was incorrectly unavailable.  

Fix: Modified validations to ensure that room availability displays correctly in this situation.

Internal Reference # 28607
(For EMS Internal Use Only)

Billing & PO Numbers

Default Billing Reference Number Empty When Prompt For Room Enabled

When users booked a room using a template that used a default billing reference number and required/validated the value, the field was empty after they selected a room as prompted.

Fix: Modified functionality so that the default value is retained when the room prompt is enabled in this situation.

Internal Reference # 27602
(For EMS Internal Use Only)

Navigation

Android: Back Button Closed Application

When users clicked the native back button from the home screen on Android devices, this closed the application rather than collapsing it. 

Fix: Modified functionality so that tapping the Android back button from the home screen collapses the application without closing it. 

Internal Reference # 27675
(For EMS Internal Use Only)

Home Page Refreshed Incorrectly When Navigating Back From Booking Details 

When users navigated back to the home page from a booking details screen, the home page did not scroll up to "Now" if the list of events extended beyond one page and they had previously navigated to see events for a specific date.

Fix: Modified functionality so that the home page refreshes to today's events when reached using the Back button. 

Internal Reference # 28532
(For EMS Internal Use Only)

Building URL Link Failed

When users clicked on Building URL from booking details screens, the link failed because the ":" character was not being carried over to the web browser.

Fix: Modified functionality so that the URL is carried over and the link functions as expected. 

Internal Reference # 28548
(For EMS Internal Use Only)
