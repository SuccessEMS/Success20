EMS Desktop Client Release Notes
V44.1, Update 3

This section describes updates to your EMS software. 

See Also: Contact Customer Support and Updating to the Latest Version of EMS (Patches and Builds).

Name: V44.1 Update 3

Release Date: August 26, 2016

Patch #:  22016.08.24.0

Build #: 44.1.253

Enhancement 

Area

Description

EMS Web App > Browse Locations > View Floor Plans

Bug 21611

Add Support for SVG File Formats for Floor Plan Images

SVG files are now supported and rendered correctly in the following areas:

  • EMS Floor Plan Configuration
  • Configuring floor map using SVG image (adding indicators, removing indicators, changing rooms, etc.)
  • EMS Web App
  • Using an SVG floor map to book space
  • EMS Desktop Client
  • Configuration > Buildings > Images tab

Fix 

Area

Description

EMS Desktop Client > New Reservation > User Defined Fields

Bug 22770

Date Conversion Miscalculated

When making a reservation using a template that had a reservation-level user defined field with a default date set, the date was converted to year-day-month format.

Fix: Changed how dates are converted for this field to always use year-month-day format.

EMS Desktop Client > Room Utilization Report

Bug 21783

Bookings Displayed on Holiday Dates

The report was showing bookings on holiday dates.

Fix: Modified the report to exclude bookings on holiday dates.

EMS Desktop Client > Email Confirmations

Bug 17233

Confirmation Emails Drop Digits to Right of Decimal

When sending email confirmations with systems that had the "Display Messages as HTML" option activated, digits to the right of decimal were missing from the emails.

Fix: Modified email triggers so that digits are no longer omitted from email confirmations in this situation.

CISCO TMS Service

Bug 17826

Clicking Retry Button Causes Error

When users selected one of the bookings in the Cisco TMS Activity window and then clicked the Retry button, the following error occurred: 

Conversion from string "Default" to type 'Integer' is not valid.

Fix: Modified the screen overlay so that the error no longer occurs and the Retry button works as expected.
