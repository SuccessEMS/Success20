Patch

Reported Issue

Issue Resolution

☒

☐

When users enabled 24 hour format on the iPad, Glance still displayed the start and end time in 12 hour format (AM, PM).

Modified functionality to check the device settings for language to determine whether to use a 12 or 24 hour clock. 

Note: This fix requires you to install the latest Glance build; we also recommend you make sure your devices use the desired clock display settings.

☐

☒

Glance displayed the booking group and event name by default. Users could designate a booking as private by configuring a status in the EMS Client. Only this single status could be used to designate a private booking.

Enhancement
​It is now possible to designate a private booking with 'Event Type' as well as status. This enhancement provides more flexibility in configuring EMS Glance.

Note: Requires latest Ems.Glance.dll and EMS Glance web installer.

Patch

Reported Issue

Issue Resolution

☒

☒

Optimization. Publishing term optimization scenarios with pre-assigned "TBD" rooms resulted in a "Scenario is invalid due to conflicts..." error even though the "TBD" room type should have allowed scheduling conflicts.

Modified validation to take into account the "TBD" room type and allow scheduling conflicts so that the error no longer presents.

Reported Issue

Issue Resolution

☐

☐

In the desktop client, users could open multiple EMS Glance Settings Windows which prevented settings from being saved.

The EMS Glance Settings in the desktop client can only be open in one window at a time. Resolution requires updated EMS.Glance.dll file and latest Desktop Client build. Updating the Glance installer is NOT required.

Patch

Issue Resolved

☒

☐

The enhancements in this build require running the latest version of the EMS Glance Web Service SQL Script (EMSGlance.sql), as well as the latest version of the EMS Glance installer (EMSGlance.msi).

☒

☐

When auto-cancellation is enabled, the Check In button and the Available bar will turn orange to better relay to the user that they need to check in before their meeting is canceled.

☒

☐

When using Authentication With Password or Authentication Without Password, the Group Lookup will now honor "Display On Web" option for that group. If the group is not DOW, it will not show up.

☒

☐

Added a new parameter that shows/hides the Event Name field on the Reserve pop-up. The Default Event Name parameter must be filled out in order to hide the Event Name field.

☒

☐

Added a new parameter for default Setup Count and another parameter that shows/hides the Setup Count field on the Reserve pop-up. The Default Setup Count parameter must be filled out in order to hide the Setup Count field.

☒

☐

Introduced a new wheel picker for Setup Count. This is currently hard-coded to display values between 0 and 25.

☒

☐

Added a new parameter to require authentication when ending or adjusting meeting.  It only applies if using Authentication With Password or Authentication Without Password.

☒

☐

UI enhancements to the Reserve pop-up. The text fields are now bigger.
