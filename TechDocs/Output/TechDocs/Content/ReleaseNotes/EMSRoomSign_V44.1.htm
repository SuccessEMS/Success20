Profile Settings

Cancel and Apply Buttons Were Not Displaying in Portrait or Landscape View

The Cancel and Apply buttons were not displaying in the EMS Room Sign App when changing profile settings.

Profile/Room Selection

Issue Selecting Profile and Room in EMS Room Sign App

Users were unable to select a profile and/or room that intersected with the time bar within the EMS Room Sign App dropdown list.

Fix: Modified functionality so that users are able to select any room and profile from the dropdown list.

(For EMS Internal Use Only)

Start/End Times

End Time Was Not Adjusting Correctly Based on Start Time

End time was not adjusting correctly based on the selected start time, and future bookings were not being created.

Fix: Modified functionality so that end time is adjusted based on selected start time.

Internal Reference # 36653
(For EMS Internal Use Only)

Create a Reservation

Unable to Create Future Bookings

Users could not book a meeting in the future.

Fix: Modified functionality so that users can book a meeting for a later time.

Internal Reference # 36654
(For EMS Internal Use Only)

Settings

Once a PIN Was Entered, the Settings Modal Did Not Save Previous Selections

After entering a PIN, the previously selected profile was not selected in the profile dropdown.

Fix: Modified functionality so that any previously saved settings populate correctly after entering a valid PIN.

Internal Reference # 36655
(For EMS Internal Use Only)

Authentication

Authentication to End Booking Was Required Even if "Authentication to End Booking" Was Set to No

EMS Room Sign App was requiring authentication to end bookings regardless of whether the setting "Require Authentication to End Booking in Progress" was set to "No."

Fix: Modified functionality so that the EMS Room Sign App does not require authentication to end booking if the parameter "Require Authentication to End Booking in Progress" is set to "No."

Internal Reference # 36657
(For EMS Internal Use Only)

Booking Details

Bookings Were Not Recognized As Private

For bookings that were set to private when the Event Type for Private Booking parameter was set to None, the private meeting label was not displaying.

Fix: Modified functionality so that the booking details display.

Room Sign

 

 

 

Using Relative URL Paths to Deploy the Room Sign App Caused a 404 Not Found Error

When deploying the Room Sign App on the cloud using relative URL paths, users were receiving a 404 Not Found error.

Fix: Modified functionality so that users can deploy the Room Sign App on the cloud using relative URL paths as long as the site uses 100% relative URL paths. It is also recommended that users avoid using paths beginning with ../

Internal Reference # 33149
(For EMS Internal Use Only)

EMS Room Sign App Error Encryption Failure

When using the Glance API with the EMS Room Sign App, the app was not functioning and users were receiving the error "Invalid Glance API URL" even though the API URL was valid. This problem stemmed from TLS 1.0 not being enabled on the server. Setting the new web.config file corrected this problem.

Fix: Modified functionality by enabling TLS 1.0 and setting the new web.config file.

Internal Reference # 33332
(For EMS Internal Use Only)

Parameters

 

New Room Sign Label Parameters

To ensure that the Room Sign App can be deployed globally, we have added several label parameters. Now all text on the Room Sign App can be customized by location. The new label parameters are:

  • End Yes buttonappears on the end meeting modal
  • Cancel buttonappears on all modals
  • OK buttonappears on all alert modals
  • Close buttonappears on booking details modal
  • Verify buttonappears on the pin modal
  • Apply buttonappears on the settings dialog

(For EMS Internal Use Only)

Fixes

Area

Description

Error Messages

 

Upgrading the Room Sign App Resulted in Error Messages on the Screen Until the Browser Was Restarted

In Update 7 we addressed a security vulnerability in Room Sign. Upgrading to any release after Update 7 caused end users to see an error message on the tablet. The browser on each tablet would have to be restarted to eliminate the error.

Fix: After upgrading to Update 15, end users will no longer see the error message.

Note: This fix is possible because Update 15 initially turns off the any anti-forgery validation.

Internal Reference # 31127
(For EMS Internal Use Only)

Kiosk Mode

 

Media Volume on Android Tablets

When the Room Sign App is run on Android tablets, EMS recommends using Kioware Lite to place the tablet in kiosk mode. The media volume was often being set very high, which resulted in end users seeing system messages on the screen.

Fix: Modified functionality so that the media volume is set to 55% when the Room Sign App is run on an Android tablet with Kioware Lite.

Internal Reference # 30315
(For EMS Internal Use Only)

Event Information

Increase Display Options for Event Information

ArmorActive Enclosure

LED Colors Stayed Orange for Checked-In And Canceled Meetings on ArmorActive Enclosure 

When users checked into a meeting that was then canceled, the LEDs on the enclosure stayed orange.

(For EMS Internal Use  Only)

Edit Reservation

Cross-listed Combo Rooms Showed As Private

When private academic courses were in combo rooms and cross-listed with other courses, they showed as "Private Meeting" when the event names should have shown in the list.

Fix: Modified functionality so that the event names now show in the list of courses and the events no longer behave as if they are private.

Internal Reference # 23867
(For EMS Internal Use  Only)

New Reservation

Automatic No-Show Incorrectly Triggered for Some Bookings

When users booked events in Glance that had no-show notifications turned on, due to a lag in saving to the database and updating Glance, some events triggered an automatic no-show status.

Fix: Modified the API so that these events update the database more efficiently and the events no longer trigger an inappropriate no-show status.

Internal Reference # 23640
(For EMS Internal Use  Only)

"Allow Future Same Day Bookings" Parameter Not Observed

When this parameter was set to No, users were still able to book future same-day bookings by double-clicking on the desired time slots on the calendar grid.

Fix:  Modified the display so that this parameter is enforced when it is set to No, clicking on the grid does not allow users to book future events.

Internal Reference # 24505
(For EMS Internal Use  Only)

Security

Web Traffic Security Vulnerabilities

The web application sent non-secure user and session information (cookies) during encrypted (SSL) sessions, exposing vulnerable information.

Fix: Added ANtiForgery validation to all Ajax HttpPost calls and added verb tampering redirection to prevent non-HTTP or Get calls being made to Glance so web traffic no longer exposes user and session information.

(For EMS Internal Use  Only)
