Configure Intents in the EMS Desktop Client

Booking Types are configured as Intents in the EMS Desktop Client. Booking Types are a new configuration available for grouping Everyday User Process templates. Administrators can configure up to five (5) Intents. Everyday User Process templates can be assigned to only one intent.

Booking Types are configured in the EMS Desktop Client. To configure an intent: 

  1. Navigate to Configuration > Everyday User Applications > Intents tab.

  1. Provide an Intent Name (there is a 20-character limit).
  2. Designate a Default Start Time.
  3. Designate a Default End Time.

NOTE: Default Start Time and Default End Time are not utilized by EMS for Google Calendar. These fields are utilized by other EMS products (e.g., EMS Mobile App).

  1. Designate a Menu SequenceThe Menu Sequence controls the order in which the Booking Types appear to users. Users see Intents within their application as a Booking Type. .
  2. Click the Everyday User Process Template tab to assign Intents to a process template. Using the Move (>) button, add all templates for your Everyday Users that match the Booking Type.

  1. Assign a priority value for these Intents. You can enter a number in the Edit Priority field or use the Move Up and Move Down buttons to set a priority relative to the other templates linked to the Intent.

NOTE: Priority is required and two process templates cannot have the same priority value. When using Intents, the Everyday User only views rooms associated with the template that has been assigned to the Booking Type.
