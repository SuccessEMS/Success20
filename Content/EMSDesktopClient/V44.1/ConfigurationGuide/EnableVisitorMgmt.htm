Enable Visitor Management

Visitors, or guest attendees who may not be part of your organization, can be added to your meetings if you enable this feature. Depending on your licensing, you may need to contact your Professional Services representative to add this feature.

To enable visitor management, you will:

  • Configure a Category for Visitors, which can be associated with booking templates
  • Set Everyday User Booking or Process Templates to allow the Visitor category
  • Add your newly created Visitor category to your Everyday User Process Templates
  • Authorize users to manage visitors

Configure a Category for Visitors

Note: The Visitor Management tab is displayed only if your organization has purchased and installed the optional Visitor Management module, which enables front-desk users such as reception to check in visitors, print badges, access lists of visitors, and run reports. The values that you specify on this tab are used when a new visitor checks into your building. See Also: Specify Unregistered Attendee/Visitor Behavior.

  1. In the , navigate to Configuration > Administration > Categories.

  1. Create a new category for visitors. From the Category tab:
    1. Enter a category name in the Category field.
    2. Ensure the Category Type field is set to Attendees.
    3. Select the Attendees Are Visitors box at the bottom of the tab to enable the Visitor checkbox as a default on your Everyday User Template.

  2. Select the Display to Everyday Users option on the Everyday User Applications tab so that the category is available for those booking templates.

  3. Add the newly created Visitors category to the Everyday User Templates to include it on the Categories tab.

  1. Authorize users to manage visitors.

Note: If you only allow certain types of users to handle visitors, be sure to assign them to the template you configured above using the Assign button. If you wish to allow visitors to check themselves in (for example, using EMS Mobile App, choose the Self Serve Mode on the Process Template tab when configuring the template, and be sure to enable it for Everyday User Applications as needed.
