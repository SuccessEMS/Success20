Configure HTTP Security Headers

When a user visits a site through a browser, the server responds with HTTP Response Headers. These headers tell the browser how to behave during communication with the site. For example, using the strict-transport-security HTTP header forces the browser to communicate solely over HTTPS. You can use the strict-transport-security and other security headers to improve web security.

For the EMS Web App, the following security headers can be configured in the web.config file:

HTTP Security Header

Description

Default

X-Frame-Options

Indicates whether a browser should be allowed to render a page in a <frame>, <iframe>, or <object>. By default, this header is disabled. To enable this header, you will need to edit the web.config file.

Note: If you want to enable the x-frame-options header and your company uses the Skype for Business Integration, the header must be configured with the value allow-from.

Disabled

X-Content-Type-Options

Blocks content sniffing by instructing the browser to follow the MIME types indicated in the header.

Enabled

Strict-Transport-Security

Forces communication using HTTPS instead of HTTP. By default, the strict-transport-security header is disabled. To enable this header, you will need to edit the web.config file.

Note: The strict-transport-security header is only valid if you are using SSL/HTTPS.

Disabled

Content-Security-Policy

Prevents attacks, such as cross-site scripting, by defining content sources that are approved and allowing the browser to load them. By default, the content-security-policy header is disabled. To enable this header, you will need to edit the web.config file.

Note: If your company uses the Skype for Business Integration and you want to enable content-security-policy, the header must be configured with the appropriate values.

Disabled
