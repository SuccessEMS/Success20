Configure HTTP Security Headers

When a user visits a site through a browser, the server responds with HTTP Response Headers. These headers tell the browser how to behave during communication with the site. HTTP Response Headers include security headers, which can improve web security. For example, using the strict-transport-security header forces the browser to communicate solely over HTTPS.

The following table lists the security headers that can be configured in the EMS Web App's web.config file.

TIP FOR EMS CLOUD CUSTOMERS: Modifications to the web.config file must be performed by EMS Cloud Operations. To request these changes, please contact EMS Support.

Security Headers Available for Configuration in EMS Web App

HTTP Security Header

Default

Description

X‑Frame‑Options

Disabled

This header indicates whether a browser should be allowed to render a page in a <frame>, <iframe>, or <object>. By default, the x-frame-options header is disabled. To enable this header, you will need to edit the web.config file.

Important: If you want to enable the x-frame-options header and your company uses the Skype for Business Integration, the header must be configured with the value allow-from.

X‑Content‑Type‑Options

Enabled

This header blocks content sniffing by instructing the browser to follow the MIME types indicated in the header.

Strict‑Transport‑Security

Disabled

This header forces communication using HTTPS instead of HTTP. By default, the strict-transport-security header is disabled. To enable this header, you will need to edit the web.config file.

Important The strict-transport-security header is only valid if you are using SSL/HTTPS.

Content‑Security‑Policy

Disabled

This header can prevent attacks, such as cross-site scripting, by defining content sources that are approved and allowing the browser to load them. By default, the content-security-policy header is disabled. To enable this header, you will need to edit the web.config file.

Important: If your company uses the Skype for Business Integration and you want to enable the content-security-policy header, you must configure this header with the appropriate values.

Configure Security Headers

To configure HTTP security headers, you will need to edit the web.config file as follows: 

TIP FOR EMS CLOUD CUSTOMERS: Modifications to the web.config file must be performed by EMS Cloud Operations. To request these changes, please contact EMS Support.

  1. Open the EMS Web App web.config file (typically, C:\inetpub\wwwroot\EMSWebApp\web.config).
  2. In the web.config file, locate the header nodes under <customHeaders>:
    <httpProtocol>
      <!-- security-specific http headers -->
      <customHeaders>
        <!--<add name="X-Frame-Options" value="deny" />--><!-- valid only when NOT using Skype 
    integration - can be configured with 'allow-from' when using Skype -->
        <add name="X-Content-Type-Options" value="nosniff"/>
        <!--<add name="Strict-Transport-Security" value="max-age=31536000"/>--><!-- valid only when
    using SSL/HTTPS -->
        <!--<add name="Content-Security-Policy" value="default-src 'self'"/>--><!-- valid only when
    NOT using Skype integration - can be configured with appropriate values when using Skype -->
      </customHeaders>
    </httpProtocol>
  3. Edit the <customHeaders> nodes according to your company's specifications and EMS Web App configuration.

    Important: Some header values are not valid with the Skype for Business Integration. Refer to the Security Header table for more information.

  4. Save the web.config file.
