fixes

Area

Description

Events List

Start and End Times Were Incorrect in Booking Details for Meetings Occurring After Daylight Saving Time Ends

For meetings that occur after Daylight Saving Time ends, the meeting start and end times were incorrect in the booking details.

Fix: Modified functionality so that when meetings occur after Daylight Saving Time ends, the meeting start and end times will display correctly in the booking details.

Internal Reference # 30910
(For EMS Internal Use Only)

Scan QR Code

After Scanning a QR Code, Selecting the "About This Location" Button Crashed the EMS Mobile App

After scanning a QR code to view a room's schedule, the EMS Mobile App would crash if users selected the About This Location button from the room’s schedule/list view.

Fix: Modified functionality so that after scanning a QR code to view a room's schedule, the About This Location page opens if users select the About This Location button from the room’s schedule/list view.  

Internal Reference # 33113
(For EMS Internal Use Only)

After Scanning a QR Code, Schedule/List View Was Shown Even Though Room Was Not Assigned to User Templates, and EMS Mobile App Crashed If Users Tried to Book the Room

After scanning a QR code for a room that was not assigned to any of the users’ templates, the room’s schedule/list view was shown, and if users selected the + button to book the room, the EMS Mobile App would crash.

Fix: Modified functionality so that when users scan a QR code for a room that is not assigned to any of the users’ templates, the EMS Mobile App will not crash; instead, the app will detect that the room is not assigned to the users’ templates, and users will receive an error message.

Internal Reference # 33115
(For EMS Internal Use Only)

After Scanning a QR Code, the Room’s Schedule/List View Did Not Load

When scanning a QR code to view a room’s schedule, the room’s schedule/list view did not load unless users re-scanned the QR code or switched between the schedule/list view.

Fix: Modified functionality so that a room’s schedule/list view loads the first time users scan the room’s QR code.

Internal Reference # 33116
(For EMS Internal Use Only)

After Scanning a QR Code, Selecting the + Button in Schedule/List View Crashed the EMS Mobile App

After scanning a room’s QR code, the EMS Mobile App would crash if users selected the + button from the room’s schedule/list view.

Fix: Modified functionality so that after scanning a room’s QR code, the app will not crash when users can select the + button from the room’s schedule/list view; instead, a list of available templates will appear when users select the + button.

Internal Reference # 33117
(For EMS Internal Use Only)

When Scanning a QR Code Associated with a Fully Defaulted Template, Users Experienced Infinite Loading When Trying to Create a Booking

When scanning a QR code associated with a fully defaulted template, users experienced infinite loading instead of having the booking created.

Fix: Modified functionality so that when users scan a QR code associated with a fully defaulted template, the booking is created.

Internal Reference # 33393
(For EMS Internal Use Only)

For Bookings Requiring Check In, Users Weren't Checked In When Scanning the QR Code

For bookings created with rooms that require check in, users were not checked in when they scanned the QR code to check in; instead, they  were shown the events list.

Fix: Modified functionality so that when a booking requires check in and users scan the QR code, users are checked in to the booking and receive a message stating that check in was successful.

Internal Reference # 33396
(For EMS Internal Use Only)

Fixes

Area

Description

Create Reservation

Contact Record Not Displayed

When creating or editing a booking, the Contact ('None') record was not displaying. 

Fix: Modified functionality so that the Contact record '(None)' is a selectable option when creating and editing a booking.

Internal Reference # 27874
(For EMS Internal Use Only)

Load More Button Displayed on the Top of Events List For 0.5 to 1 Second After Creating and Saving Booking

When users were creating or editing a reservation, the Load More button displayed at the top of the Events List for only 0.5 to 1 second after creating and saving the booking. 

Fix: Modified functionality so that the Load More button doesn't appear at the top of the Events List right after creating and saving the booking.

Internal Reference # 30948
(For EMS Internal Use Only)

User Defined Field

User Defined Fields Were Unreadable Due to Truncation

When a template with a user defined field (UDF) was selected, users were unable to read the UDF question in its entirety. 

Fix: Modified functionality so that users can read the UDF in order to provide an intelligent answer. 

Internal Reference # 30892
(For EMS Internal Use Only)

Events List

Check In Icon Not Displayed After Checking In

The Check In icon was not displaying right after the user checked in. User was forced to manually refresh the Events List. 

Fix: Modified functionality so that the Events List automatically refreshed and the Check In icon appeared near the booking. 

Internal Reference # 31009
(For EMS Internal Use Only)

After Canceling a Booking from the Drawer, the Spinning Circle Disappeared

The spinning circle was missing after canceling a booking from the drawer. 

Fix: Modified functionality so that the spinning circle is present before booking is canceled and the Events List updated. 

Internal Reference # 31039
(For EMS Internal Use Only)

Skype for Business

Mobile App Crashed When User Clicked "Continue Without Adding Skype" Button

If Skype is not configured correctly, clicking the "Continue Without Adding Skype" button crashes the app instead of leading to Create Reservation page. 

Fix: Modified functionality so that users clicking Continue Without Adding Skype button are led to the Create Reservation page. 

Internal Reference # 31068

Previously Entered UDFs Were Refreshing After Enabling Skype Toggle

When the Skype toggle was enabled for the first time, previously entered user-defined fields (UDFs) were refreshing. 

Fix: Modified functionality so that previously entered UDFs do not refresh and users do not have to repopulate UDFs.

Internal Reference # 31327
(For EMS Internal Use Only)

Configuration

Mobile App Crashed When Users Imported OpenID Configuration

When users configured the EMS Mobile App to use OpenID, the config import caused the app to crash. The app would continue to crash after each subsequent try until the user uninstalled and then reinstalled the app. 

Fix: Modified functionality so that the OpenID configuration import does not cause the EMS Mobile App to crash. 

Internal Reference # 31093
(For EMS Internal Use Only)

Enhancements

Area

Description

Skype for Business

User Authentication for Skype for Business Online

When users toggle Skype for Business for the first time, they will be presented with a Skype for Business authentication form. If users successfully authenticate, a valid token will be generated and will not be required to authenticate again when adding Skype for Business to bookings in the EMS Mobile App. If users fail to authenticate, the Skype for Business toggle will be disabled. 

Internal Reference # 29610
(For EMS Internal Use)

User Authentication for Skype for Business On Premise

When users toggle Skype for Business for the first time, they will be presented with a Skype for Business authentication form. If users successfully authenticate, a valid token will be saved for 8 hours. If users fail to authenticate, the Skype for Business toggle will be disabled.

Internal Reference # 29770
(For EMS Internal Use)

Add a Skype Meeting to an EMS Reservation

Users can add Skype for Business to their reservation by using the Skype Meeting toggle on Exchange-enabled templates.

Internal Reference # 28890
(For EMS Internal Use)

Skype for Business Meeting Details and a Join Skype Meeting Link Appear in Reservation Details

When Skype for Business is added to a reservation, booking details will include a Skype link with dial-in details and a Join button. A Join Skype button appears next to the reservation in the Events List. 

Internal Reference # 28891
(For EMS Internal Use)

Skype for Business Meeting Link Included in Email Notifications to Attendees

When Skype for Business is added to a reservation, a link to the Skype Meeting—including dial-in details—will be included in email notifications sent to attendees. 

Internal Reference # 28892
(For EMS Internal Use)

Skype for Business Calendar Updates

When Skype for Business is added to a reservation, booking details—such as time, date, description, Join Skype link, etc.—will be added to attendee's and organizer's Skype calendars.

Internal Reference # 28893
(For EMS Internal Use)

Join a Skype for Business Meeting from the Meeting Link That Appears on the Mobile App

Users can join a Skype for Business Meeting by either left-swiping the meeting from the Home page or clicking the Join icon in the meeting details. 

Internal Reference # 28894
(For EMS Internal Use)

Editing a Reservation with Skype for Business

Users can edit their reservation (date, time, location, etc.) but once they have added Skype to their reservation, they cannot remove it. Users can delete the link but the link will still be active.

Internal Reference # 28895
(For EMS Internal Use)

Canceling a Booking with Skype for Business

When users cancel a meeting that includes Skype for Business, the changes will be reflected on the user's Outlook calendar.

Internal Reference # 28896
(For EMS Internal Use)

Area

Description

UDF Configuration

Create or Edit Booking Required/Not Required Flag Were Not Being Honored

Child UDFs are required or optional based on the actual Required or Not Required flags being set when configuring the child UDF. 

Fix: Modified functionality to ensure all types of child UDFs respect the Required/Not Required flag in the UDF configuration. 

Internal Reference # 27910
(For EMS Internal Use)

Search

"Show All Areas" Search Function Was Not Filtering Building Correctly

When a user selected the Show All Areas search function, no search results were returned. 

Fix: Modified functionality so that search works correctly regardless of the selected area. 

Internal Reference # 28501
(For EMS Internal Use)

About This Location

From the "About This Location" Screen, Hitting Back Button Led to QR Scanner

Hitting the back button from the About This Location screen should have brought users to the List View/Room Schedule screen but instead led to the QR code scanner. 

Fix: Modified functionality so that clicking the native back button results in the application opening the List View/Room Schedule screen.

Internal Reference # 28513
(For EMS Internal Use)

Edit Booking

Editing Rooms With Pricing Resulted in Error

When a user was editing a booking, clicking on the Editing Room with Pricing or switching to a room with pricing during booking resulted in an error.

Fix: Modified functionality so that users can edit or switch to a room with pricing. 

Internal Reference # 28427
(For EMS Internal Use)

Images

Images in Building/Room Details Not Displaying on Android Phones

Android phone users cannot view .gif images on the Building or Room Details screens. 

Fix: Modified functionality so that Android users can view .gif file images. 

Internal Reference # 27942
(For EMS Internal Use)

About Page

API Revision Number Missing on About Page

User Authentication Issues Due To Custom config.json File
