Fixes

Area

Description

Recurring Reservations

EMS for Outlook Add-in Was Not Correctly Updating the Location Field

The EMS for Outlook add-in was not correctly updating the Location field in the Outlook meeting window when room changes were made to recurring reservations. When room changes were made in the EMS for Outlook add-in, the Location field would display the previous (old) room name instead of just displaying the updated room name.

Fix: Modified functionality so that the EMS for Outlook add-in updates the Location field with the appropriate room name(s).

Internal Reference # 35903 and 36738
(For EMS Internal Use Only)

Resolve Conflicts

Resolve Conflict Window Did Not Appear in Microsoft Outlook 2013

For Microsoft Outlook 2013 users who were acting as delegates, the Resolve Conflict window did not appear when conflicts occurred.

Fix: Modified functionality so that the Resolve Conflict window appears when conflicts occur.

Internal Reference # 39013
(For EMS Internal Use Only)

Opening Add-in

Users Received an Error Message That They Were Not Configured As Their Own Delegate

When trying to access the EMS for Outlook add-in, some users were receiving an error message stating that they were not configured as their own delegate. These users could not open and use the Outlook add-in.

Fix: Modified functionality so that users no longer receive an error message and can use the EMS for Outlook add-in.

Internal Reference # 39902
(For EMS Internal Use Only)

Canceling a Reservation

Reservation Canceled in EMS for Outlook Add-in Not Canceled in EMS Desktop Client

When users canceled a reservation in the EMS for Outlook add-in, the reservation was not canceled in the EMS Desktop Client.

Fix: Modified functionality so that when users cancel a reservation in the EMS for Outlook add-in, the reservation is also canceled in the EMS Desktop Client.

Internal Reference # 39942
(For EMS Internal Use Only)

Create Reservations

EMS for Outlook Add-in Did Not Show Correct Selected Time for Indian Standard Time in the Schedule View

When creating reservations in Indian Standard Time (IST), buildings from non-IST time zones did not show the correct selected time in the Schedule View of the EMS for Outlook add-in. Additionally, when creating reservations in non-IST time zones, buildings from the IST time zone did not show the correct selected time in the Schedule View.

Fix: Modified functionality so that when Indian Standard Time is selected for creating reservations, non-IST buildings show the correct time in the Schedule View, and when a non-IST time zone is selected for creating reservations, the IST buildings show the correct time in the Schedule View.

Internal Reference # 39981
(For EMS Internal Use Only)

Error Initializing the EMS for Outlook Add-in

Some users were unable to make reservations because there was an error initializing the EMS for Outlook add-in when everyday user names and/or group names contained extra commas.

Fix: Modified functionality so that everyday user and group name formats that contain apostrophes or commas are valid, and users are able to make reservations in the Outlook add-in with these everyday user names and group names.

Internal Reference # 40011
(For EMS Internal Use Only)

Location Field and Exchange Room Integration (ERI) Email Were Not Updating Correctly When Changes Were Made to a Reservation Before Saving

The Location field and the email address for an Exchange Room Integration (ERI) were not updating correctly when room changes were made prior to saving a reservation. In the Microsoft Outlook meeting window, the room name for the original room would remain in the Location field, and the ERI email address would disappear from the To field.

Fix: Modified functionality so that the Location field and ERI email address update correctly when changes are made to a reservation prior to saving.

Internal Reference # 40045
(For EMS Internal Use Only)
