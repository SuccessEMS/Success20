Pending Status for Returning EMS Web App Users

For EMS Web App users who were reactivated through HR Toolkit and configured to have active status on login, these users were given a pending status, instead of active, when they logged into the EMS Web App. Furthermore, EMS administrators had to manually change these everyday user account statuses from pending to active.

Fix: Modified functionality so that there is a parameter to set the status of returning users to either Active or Pending: 

  • If set to Active, then users are activated when they log into the EMS Web App.
  • If set to Pending, an account pending message will display when users log into the EMS Web App.

Unable to Create Reservations with Catering Services Added

For customers using F5 BIG-IP® Access Policy Manager with a policy to not accept host header information, users were unable to create reservations with catering services added.

Fix: Modified functionality so that users can create reservations with catering services added.
