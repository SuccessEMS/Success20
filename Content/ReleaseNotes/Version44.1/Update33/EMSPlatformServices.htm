Area

Logging

"Always On" Detailed Logging for Error Requests

When an error occurs during a request, the complete set of log entries for the request are now captured and outputted as follows:

  • On a successful request, the platform will only output logs at the default or specified log level (e.g. "Warning").
  • On an erroneous request, the platform will output fine-grained logs for that request (e.g. all DEBUG/INFO log entries).

EMS Internal Reference # 35085
(For EMS Internal Use Only)

Logs

Enhanced Log Filtering Functionality

Raw and formatted logs can now be filtered by date and time in the EMS Platform Services Admin Portal. This enhancement will facilitate troubleshooting.

EMS Internal Reference # 40234
(For EMS Internal Use Only)

SAML Authentication

 

SP Metadata URL Displayed

For the convenience of users configuring SAML authentication, a field has been added to display (but not edit) the metadata URL in the EMS Platform Services Admin UI.

EMS Internal Reference # 38092
(For EMS Internal Use Only)

 

Metadata Import

The SAML configuration screen allows users to import IdP metadata by file upload and by URL in the EMS Platform Services Admin UI. The file mechanism will use the existing API route to upload; however, the URL mechanism needs an alternative route.

EMS Internal Reference # 38093
(For EMS Internal Use Only)
 

Parameter Events 2HVAC Added

Events2HVAC — Get In Progress Bookings — parameter added so that users can get bookings that start OR end within filtered dates that includes "in progress" bookings.

EMS Internal Reference # 29337
(For EMS Internal Use Only)

For custom integrations on the EMS Platform that had logging enabled, custom integration logs were being combined with global logs, but with missing information.

Booking Information Changes Missing in History Tab

Fix: Modified functionality so that changes to booking's critical information (e.g., date, time, room, attendee, etc.) are captured in the booking's history tab in the EMS Desktop Client.
