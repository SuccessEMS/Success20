EMS Dashboard

 

Notification Timezone Issues

Reservation notifications on the EMS Dashboard were reflecting the server timezone, and not the timezone where the user created/was viewing the reservation.

Fix: Modified functionality so that the timezone displayed on the EMS Dashboard is the timezone where the user created/viewed the reservation.

Internal Reference # 30419
(For EMS Internal Use Only)

Audit Time Stamps Reflected Incorrect Time

Audit time stamps in the EMS Dashboard and Manage Services were reflecting the local time of the SQL server, instead of the local time of the user.

Fix: Modified functionality so that the dateTime should reflect the time set on the user's machine.

Internal Reference # 34510
(For EMS Internal Use Only)

Authentication

LDAP Authentication Failed if Advanced Communication Enabled

Authentication to the EMS Desktop Client using LDAP did not work when advanced communication options were enabled.

Fix: Modified functionality so that users can authenticate to the EMS Desktop Client using LDAP with or without advanced communications options enabled.

Internal Reference # 41041
(For EMS Internal Use Only)

Debugging Errors

 

Just In Time (JIT) Debugging Errors

Users closing multiple different GUI Windows in the EMS Desktop Client experienced JIT debugging errors. When JIT debugging was enabled, any unhandled exception was sent to the JIT debugger registered on the computer instead of the dialog box.

Internal Reference # 40794
(For EMS Internal Use Only)

Area

Description

Billing Reference

PO Numbers

Group Default Billing Reference and PO Numbers Were Not Populating

When creating a new reservation in EMS for Outlook on a template that displays the Billing Reference and PO numbers, these default values were not populating. Users had to toggle to a different group and back to the original group to populate the defaults.

Fix: Modified functionality so that the default BR and PO values for a group are populated in the respective fields by default when creating a new reservation on a template that displays these fields.

Internal Reference # 40968
(For EMS Internal Use Only)

Web Deploy

Web Deply for EMS for Outlook Experienced Crashing and Memory Leak

Users making reservations using the Web Deploy of EMS for Outlook experience a memory leak and machine crashing.

Fix: Modified functionality so that the Web Deploy of EMS for Outlook did not crash or cause memory leaks.

Internal Reference # 40993
(For EMS Internal Use Only)

Error Messages

Date Incorrect on Error Message

Regardless of what was configured in the "Allow New Bookings Within Range of this Many Days," users experienced the error message "You cannot book this Room after XX/XX/XXXX. . . ." The date on the error message was one day before the last allowed booking date.

Fix: Modified functionality so that the error message reflects the correct date.

Internal Reference # 41099
(For EMS Internal Use Only)
