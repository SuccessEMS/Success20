Create or Edit Reservation

Error After Creating a Reservation in the EMS for Outlook Add-in

For some EMS for Outlook add-in users, when they used a template to create a reservation, an error occurred after clicking Save &amp; Close or Send, and the reservation was not saved in the EMS database. However, the invite was sent to attendees.

Fix: Modified functionality so that when users create a reservation in the EMS for Outlook add-in, the template information populates appropriately, and the reservation is created with no errors.

Internal Reference # 36800
(For EMS Internal Use Only)
 

Combo Room Components Showed Incorrectly in EMS for Outlook

When reserving a combo room in the EMS for Outlook add-in, the Schedule View and List View showed the combo room as unavailable once the combo room was selected; however, the component rooms were displayed as available (+ symbol showed next to the component rooms).

Fix: Modified functionality so that Schedule View and List View show the component rooms as unavailable if the parent combo room has been selected.

Internal Reference # 39620
(For EMS Internal Use Only)

Conflict Resolution and
Recurring Reservations

EMS for Outlook Add-in Was Unresponsive When Users Tried to Resolve Conflicts

When resolving booking conflicts for a recurring meeting, the select room button (+) on the Resolve Conflicts page was unresponsive, and users were unable to select a new location to resolve the conflict.

Fix: Modified functionality so that the select room button (+) on the Resolve Conflicts page functions properly, and users can select a new location to resolve a conflict.

Internal Reference # 40727
(For EMS Internal Use Only)
Billing Reference and PO Numbers

Group Default Billing Reference (BR) and PO Numbers Were Not Populating

When creating a reservation in EMS for Outlook, the BR and PO values were not populating by default in their respective fields. The defaults were populating when users toggled from a different group back to the original group.

Fix: Modified functionality so that the default BR and PO values for a group are populated in the respective fields by default when creating a new reservation on a template that displays these fields.

Internal Reference # 40968
(For EMS Internal Use Only)
 

Incorrect Time Bar in Attendee Schedule View

Time labels across the attendees area of schedule view was not reflecting the correct time for the organizer and attendees.

Fix: Modified functionality so that the attendee schedule view reflects the correct time.

Internal Reference # 41582
(For EMS Internal Use Only)
Create a Reservation

Incorrect Time Bar in Attendee Schedule View

Regardless of what was configured in the "Allow New Bookings Within Range of this Many Days," users experienced the error message "You cannot book this Room after XX/XX/XXXX. . . ." The date on the error message was one day before the last allowed booking date.

Fix: Modified functionality so that the correct last allowed booking date displays if users try to book beyond that date.

Internal Reference # 41099
(For EMS Internal Use Only)
Recurring Reservation

Send Update Button Not Updating Database

EMS for Outlook was not updating the time in the Database when users dragged/dropped a single occurrence of a recurring reservation and used the EMS for Outlook Send Update button, unless the time was changed in the meeting dropdown. This issue also occurred for single reservations; however, there was no delay using the native Outlook Send Update button.

Fix: Modified functionality so that dragging and dropping alone in EMS for Outlook updates the time in the database.
