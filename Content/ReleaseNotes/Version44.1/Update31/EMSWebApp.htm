Setup Type / Count Consistency

Updated Error Message When Attendance Meets Default Setup Type But Meets Selected Setup Type

Messaging has been updated to help the user better understand Setup Type and Count messages when searching and why they may not be able to proceed (e.g., entering a set-up count that does not meet the specific selected setup type even though it meets the default that is displayed int he Schedule Grid.)

Cancellations

Cancel All-Bookings for A Date Simultaneously on VC Template

Cancel flow has been updated so that users can now cancel all of the bookings on a single date instead of having to cancel each room individually.

Internal Reference # 41204
(For EMS Internal Use Only)

Video Conference (VC) Templates

"New Booking" Added to VC Templates

New Booking button has been added to non-calendar VC templates.

Internal Reference # 41219

User-defined Fields (UDFs)

Add Optional Reservation-Level UDFs from Reservation Edit

When editing Additional Information from the Edit Reservation page, users can now complete optional UDFs that they have opted to skip during the initial Create process.

Fixes

Area

Description

BR/PO Numbers

Group-level Billing Reference /PO Number Search Not Functioning

When attempting to select a Group from the Reservation Details page, default BR/PO numbers were not pre-populating and the type-ahead search function did not produce a cumulative list of BRs/POs assigned to that Group and/or globally.

Fix: Modified functionality so that users can search for BR/PO numbers.

Internal Reference # 37212, 41189
(For EMS Internal Use Only)

Availability Tab

Parameter Value Not Reflected on Schedule Grid on Availability Tab

When the parameter "Event Information to Display in Booking on Schedule View" was changed to something other than Event Name (e.g., Group Name, None), Event Name was still displayed in the Schedule Grid on the Availability tab.

Fix: Modified functionality so that the Availability tab respects the parameter value configured for "Display in Booking on Schedule View."

Internal Reference # 41059
(For EMS Internal Use Only)

Video Conference (VC) Reservations

VC Host improperly Identified When Added as Conflict-Exception in Recurrence

When there was a conflict exception on the second location added in a recurring meeting, the second location was duplicated as the VC host on the excepted dates.

Fix: Modified functionality so that there is only one host on a VC recurrence, even with exceptions.

Internal Reference # 41711
(For EMS Internal Use Only)

Room Search

'I Know What Room I Want' Search Not Functioning

When searching for a room utilizing the "I Know What Room I Want" function, the search results did not return all rooms that matched the search criteria.

Fix: Modified functionality so that the I Know What Room I Want search function returns the correct list of rooms.

Internal Reference # 41712
(For EMS Internal Use Only)

Attendance Count

Incorrect Attendance Count for Second Location

When adding multiple locations to a reservation, the attendance per booking reflected the attendee count of the first location (regardless of if you indicated a different count for the second location).

Fix: Modified functionality so that the attendance for the second location is accurately recorded.

Internal Reference # 41714
(For EMS Internal Use Only)

G Suite

Google Calendar Recurrence Issues

In certain circumstances, when creating a Google-enabled recurrence from the EMS Web App, the recurrence would not be reflected on the Google calendar of the host and guests.

Fix: Modified functionality so that the Google-enabled reservations are always reflected on the host and guest calendars.

Internal Reference # 42096
(For EMS Internal Use Only)

Create a Reservation

Unable to Remove Filters When Creating a Reservation

When accessing the EMS Web App on a tablet, users could select filters (Feature, Building, Room, and Type), but not deselect them, when creating reservations.

Fix: Modified functionality so that users can deselect filters.

Internal Reference # 41023
(For EMS Internal Use Only)
 

When Default View Was List, No Rooms Were Found for Default Building

When a default building was specified and the default view was List, no rooms were returned in the search.

Fix: Modified functionality so that rooms can be found when searching a default building.

Internal Reference # 42269
(For EMS Internal Use Only)
EMS Platform Services

EMS Platform Services Communication Failure

When TLS 1.1 was enabled on the web server, communication failed and generated an error message.

Fix: Modified functionality so that users do not receive an error message when TLS 1.1 is enabled.

Internal Reference # 42731
(For EMS Internal Use Only)
 

Billing Reference and PO Number Failed to Update When Group Was Changed

When editing reservation details on a reservation with a Group change, the Billing Reference and PO numbers were not updated.

Fix: Modified functionality so that Billing Reference and PO numbers update when a group changes.

Internal Reference # 42731
(For EMS Internal Use Only)
 

Billing Reference Error When Editing Reservation

When Billing Reference configuration was set to Required and Validated and not set to Show, a Billing Reference Required error displayed when editing a reservation.

Fix: Modified functionality so that users do not experience a billing reference error.

Internal Reference # 41061
(For EMS Internal Use Only)
 

No Communication Between EMS Reservations and Google Calendar

When a reservation is created in EMS, the reservation does not appear in Google Calendar.

Fix: Modified functionality so that reservations created in EMS appear in Google Calendar.

Internal Reference # 41124
(For EMS Internal Use Only)
 

SAML Error

When logged in with Okta SAML credentials, users experienced a 404 error.

Fix: Modified functionality so that users do not received 404 errors.

Internal Reference # 42520
(For EMS Internal Use Only)
