Fixes

Area

Description

BR/PO Numbers

Group-level BR/PO Number Search Not Functioning

When attempting to select a Group from the Reservation Details page, default BR/PO numbers were not pre-populating and the type-ahead search function did not produce a cumulative list of BRs/POs assigned to that Group and/or globally.

Fix: Modified functionality so that users can search for BR/PO numbers.

Internal Reference # 37212
(For EMS Internal Use Only)

Availability Tab

Parameter Value Not Reflected on Availability Tab

When the parameter "Event Information to Display in Booking on Schedule View" was changed to something other than Event Name (e.g., Group Name, None), Event Name was still reflected on the Availability tab.

Fix: Modified functionality so that the Availability tab respects the parameter value configured for "Display in Booking on Schedule View."

Internal Reference # 41059
(For EMS Internal Use Only)

Video Conference (VC) Reservations

VC Host improperly Identified When Added as Conflict-Exception in Recurrence

When there was a conflict exception in a recurring meeting, the VC host was displaying as both locations.

Fix: Modified functionality so that the correct location is displayed as the VC host.

Internal Reference # 41711
(For EMS Internal Use Only)

Room Search

'I Know What Room I Want' Search Not Functioning

When searching for a room utilizing the "I Know What Room I Want" function, the search results did not return all available rooms.

Fix: Modified functionality so that the I Know What Room I Want search function returns the correct list of available rooms.

Internal Reference # 41712
(For EMS Internal Use Only)

Attendance Count

Incorrect Attendance Count for Second Location

When adding a room to a reservation with two locations, the attendance per booking reflected the attendee count of the first location.

Fix: Modified functionality so that the attendance for the second location is accurately displayed.

Internal Reference # 41714
(For EMS Internal Use Only)
G Suite

G Suite Recurrence Create

For Google-enabled recurring meetings with a start date in the future, the EMS Web App was not sending the appropriate start/end times to EMS Platform Services.

Fix: Modified functionality so that appropriate start/end times are sent to the EMS Platform from the EMS Web App.

Internal Reference # 42096
(For EMS Internal Use Only)
