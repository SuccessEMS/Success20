Internal Reference # 37851
(For EMS Internal Use Only)

Reservation Details

Access to Browse Reservation Details

Available through Configuration > Everyday User Security Templates, the new security role allows Administrators to configure unauthenticated access for viewing reservation summary details.

On the (unauthenticated user) Everyday User Security Template, authentication is not required to view the ReservationSummary page. If not selected, viewing ReservationSummary requires an authenticated session of the EMS Web App. For all other Everyday User Process Templates, the user has access to view the ReservationSummary page. If not selected, only Owner or Delegate of the meeting can view all details of the ReservationSummary page. All other authenticated users will have access to the unauthenticated view of the ReservationSummary page.

See Also: EMS Desktop Client Release Notes Update 26.

Fixes

Area

Description

Edit Reservation Page

Add Booking Button Was Appearing on Video Conference Template

Users were briefly seeing the Add Booking button on the Edit Reservation page in the EMS Web App.

Fix: Modified functionality so that the Add Booking button does not ever display on a VC template or Calendar-enabled template.

Internal Reference # 36849
(For EMS Internal Use Only)

LDAP Authentication

 

Intermittent Slowness Experienced During Authentication (EMS Cloud Services Customers)

EMS Cloud Services customers experienced intermittent slowness when authenticating with LDAP.

Fix: Modified functionality to eliminate any latency during LDAP authentication for EMS Cloud Services customers.

Internal Reference # 38324
(For EMS Internal Use Only)

Create a Reservation

Users Could Request Parts of and Whole Combo Rooms

When Requesting Rooms, users were able to select a combo room and its components without receiving a warning message regarding conflicts.

Fix: Modified functionality so that users cannot select the individual component rooms within that combo room and will receive an error message.

Internal Reference # 36802
(For EMS Internal Use Only)

Responsive State

Hamburger Menu Functionality

When viewing the EMS Web App on a small screen in responsive state, the hamburger menu was auto expanding. This made working with the event impossible.

Fix: Modified functionality so that the hamburger menu does not auto expand and users can access their events on the EMS Web App in a responsive state.

Internal Reference # 36862
(For EMS Internal Use Only)

Create a Reservation

Booking Rule Error

EMS Web App users received a booking rule violation error when booking a building that is closed on the day of request.

Fix: Modified functionality so that users can create a reservation on a day the building is closed if the booking complies with all booking rules.

Internal Reference # 37663
(For EMS Internal Use Only)

Browse Locations

Reverted Date in Browse Locations

Users were experiencing a reverted date when hitting enter in Browse Locations if the cursor was in the Find a Room search field.

Fix: Modified functionality so that users can complete their search without the date changing.

Internal Reference # 38550
(For EMS Internal Use Only)

User-Defined Fields (UDFs)

Required UDFs Were Missing from EMS Web App Reservations

Dependent required user-defined fields (UDFs) that were required were not appearing on reservations made in the EMS Web App.

Fix: Modified functionality so that required UDFs appear on reservations made in the EMS Web App.

Internal Reference # 38716
(For EMS Internal Use Only)

Search Functionality

Latency with GetAvailability and LookupAttendee Searches

Fix: Existing attendee lookup calls in the EMS Web App have been removed to prevent any latency.
