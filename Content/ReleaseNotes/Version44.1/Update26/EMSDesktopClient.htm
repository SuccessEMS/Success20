New Security Role Added to EMS Desktop Client: View Reservation Summary

Available through Configuration > Everyday User Security Templates, the new security role allows Administrators to configure unauthenticated access for viewing reservation summary details.

On the (unauthenticated user) Everyday User Security Template, authentication is not required to view the ReservationSummary page. If not selected, viewing ReservationSummary requires an authenticated session of the EMS Web App. For all other Everyday User Process Templates, the user has access to view the ReservationSummary page. If not selected, only Owner or Delegate of the meeting can view all details of the ReservationSummary page. All other authenticated users will have access to the unauthenticated view of the ReservationSummary page.

Email Notifications Were Not Being Sent

When reservations were created with bookings that met the conditions of the Group Notification rule, group email notifications were not being sent.

Fix: Modified functionality so that Everyday Users receive the group notification.
