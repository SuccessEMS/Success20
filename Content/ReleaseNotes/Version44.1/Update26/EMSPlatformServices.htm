API Calls

Edit In-Progress Bookings

Modified the PATCH/bookings/{id} route to allow editing the Start Time and End Time fields for a booking that is in-progress:

Note that in-progress edits will NOT be supported on the following booking types:

  • Bookings with service orders
  • Bookings with setup/teardown times
  • Video conference bookings
  • Integrated calendar bookings (Microsoft Exchange and G Suite)
  • Managed space (request rooms) bookings

For this enhancement, no changes were made to the notifications that are triggered on changing the start time and end time.

Logging

EMS Platform Services Version Included in Logs

To aid in remote customer debugging, the logging mechanisms have been updated to include the EMS Platform Services version in the logs.

EMS Internal Reference # 34897
(For EMS Internal Use Only)

"Always On" Detailed Logging for Error Requests

When an error occurs during a request, the complete set of log entries for the request are now captured and outputted as follows:

  • On a successful request, the platform will only output logs at the default or specified log level (e.g. "Warning").
  • On an erroneous request, the platform will output fine-grained logs for that request (e.g. all DEBUG/INFO log entries).

EMS Internal Reference # 35085
(For EMS Internal Use Only)
