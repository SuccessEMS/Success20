Fixes

Area

Description

Booking Rules

DaysAvailable Returned With Incorrect Value

When the range of days booking rule is applied, the daysAvailable value being returned was incorrect.

Fix: Modified functionality so that the booking rule is honored and the daysAvailable value is correct.

Internal Reference # 34947
(For EMS Internal Use Only)

Check-In

Web User Check-In Functionality Broken

For customers who upgraded to Update 22, web user-based check-in on bookings failed and users received an error.

 

Fix: Modified functionality so that web users can check-in to bookings.

Internal Reference # 37054
(For EMS Internal Use Only)

UDFs

Searching for Multiple Values of the Same UDF Returned Partial Results

Users searching for multiple values of the same UDF experienced only partial results (i.e., results for which ever key value was last in the list).

Fix: Modified functionality so that searching on multiple key values should return all matching results.

Internal Reference # 37068
(For EMS Internal Use Only)

UDF Search Conflict

UDFs with matching names, but defined on different parent types, were conflicting in UDF searches.

Fix: Modified functionality so that there is no conflict while searching UDFs.

Internal Reference # 37100
(For EMS Internal Use Only)

EMS Mobile App

API Call Panicked When Creating Skype Reservation from EMS Mobile App

The POST/reservations/actions/create call panicked when creating a Skype reservation from the EMS Mobile App.

Fix: Modified functionality so that the POST/reservations/actions/create call works appropriately when creating a Skype reservation from the EMS Mobile App.

Internal Reference # 37076
(For EMS Internal Use Only)
