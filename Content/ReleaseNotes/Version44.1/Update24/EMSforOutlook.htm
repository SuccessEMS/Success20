Create or Edit Reservation

Microsoft Outlook 2010 (64x) Crashes on Save & Close from the EMS for Outlook Add-in

For customers using Microsoft Outlook 2010 (64x), when users tried to Save & Close an appointment from the EMS for Outlook add-in, Microsoft Outlook crashed.

Fix: Modified functionality so Microsoft Outlook does not crash when users click Save & Close in the EMS for Outlook add-in.

Internal Reference # 33452
(For EMS Internal Use Only)

Recurring Reservations

 

Setup Type and Attendee Count Fields Not Retaining Original Booking Details Values After Resolving Conflicts in a Recurring Reservation

When users modified the meeting time and resolved conflicts in a recurring reservation, the Setup Type and Attendee Count fields lost their original booking details values, and users needed to reselect these values on the original room.

Fix: Modified functionality so that when users modify the meeting time and resolve conflicts in a recurring reservation, the Setup Type and Attendee Count fields retain their original booking details values.

Internal Reference # 34688
(For EMS Internal Use Only)

Combo Rooms
Recurring Reservations

Combo Room Name Displayed in Location Field After Component Room Had Been Removed from a Recurring Reservation

When users removed a component room and added a new room in a recurring reservation, the Location field displayed the combo room name.

Fix: Modified functionality so that the combo room name is removed from the reservation and does not display in the Location field.

Exchange Integration Web Service

Poor Performance with Platform Services (EMS for Outlook Calls)

testing performance of the outlook plugin. (Via calling platform services)

The last test resulted in a spike in response times about halfway through the test. The operations team was involved with this test and Bill M wanted to have a SF case created so activity can be tracked.

**Requested full report from Devin instead of the below screenshot**

In order to improve the performance of the Exchange Integration Service under high load the logging framework is now logging events asynchronously to eliminate thread contention when logging to a single file. Additionally new json properties have been added to the logging to make it easier to query results from splunk.

To prove that nothing has been broken in the recent changes the following should be tested:

Web App:

GetUserAvailability (Exchange Availability Schedule View)

LookupAttendees (autocomplete)

Create/Edit/Cancel Meetings (verify exchange is updated)

Manage Attendees (use the web app manage attendees option)

Outlook:

Create Outlook Appointment (Platform calls PAM to Convert the Outlook ID to EWS ID)

Add Attendees to Schedule View (Outlook calls platform which calls PAM to lookup attendees and request user availability)

Splunk logs can be viewed by using queries similar to the following:

index=int customer=legal app=Ems.Exchange.Integration
