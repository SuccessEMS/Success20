Fixes

Area

Description

Create or Edit Reservation

Microsoft Outlook 2010 (64x) Crashes on Save & Close from the EMS for Outlook Add-in

For customers using Microsoft Outlook 2010 (64x), when users tried to Save & Close an appointment from the EMS for Outlook add-in, Microsoft Outlook crashed.

Fix: Modified functionality so Microsoft Outlook does not crash when users click Save & Close in the EMS for Outlook add-in.

Internal Reference # 33452
(For EMS Internal Use Only)

Recurring Meetings

 

Outlook Add-in Recurring meeting changing time and resolving conflict - set up type and attendance not holding original booking details

For a recurring meeting, if users modified the meeting time and resolved conflicts, the Setup Type and Attendee Count fields lost their original booking detail values, and users would need to reselect these values on the original room.

Fix: Modified functionality so that when users modify the meeting time and resolve conflicts for a recurring meeting, the Setup Type and Attendee Count fields retain their original booking detail values.

Internal Reference # 34688
(For EMS Internal Use Only)

Combo Room Issue

 

Recurrence Outlook Reservation Reflecting Combo Room Name in Location Field after Component Room has been removed from the reservation

Combo Room name is displayed in the Location Field.

Expected Results: Combo Room should be removed from all aspects of the Reservation, and should not be displayed

 

Fix: Modified functionality so that the combo room name is completely removed from the reservation and does not display in the Location field.

Internal Reference # 36873
(For EMS Internal Use Only)
 

Combo Rooms Not Booking Correctly

When booking combo rooms in the EMS for Outlook Add-in and adjusting the time, the combo room does not appear.

Fix: Modified functionality so that the booking time is updated and rooms display as booked.

Internal Reference # 36740
(For EMS Internal Use Only)
