Attendee Search

 

Configuration for Search on Type Ahead Fields

The parameter Delay for Type Ahead Search Trigger (ms) now affects all type-ahead search fields (note that in Update 26, the parameter only affected the Find Attendee field on calendar-enabled templates, Exchange and G Suite). The parameter lets administrators configure the duration after the last keystroke entry in a search field before triggering a search call. The default is 400 ms.

See Also: EMS Desktop Client U26 Release Notes and Everyday User Applications Parameters

Fixes

Area

Description

Browse Events

Events Were Missing from Browse Events

If events took place in locations that were not included in the unauthenticated web process template, these events would not display for authenticated users in Browse Events, even though the Event Type, Status, and Room settings were set to Display to Everyday Users.

Fix: Modified functionality so that when an event is configured with the Event Type, Status, and Room settings set to Display to Everyday Users, the event will appear in Browse Events.

Internal Reference # 33741
(For EMS Internal Use Only)

Create a Reservation

Prematurely Displayed Pop-up "Attendance must be greater than zero with rooms that have a default Setup Type"

If the parameter "Setup Type Validation Rule" was set to "show and validate," users who selected a room with the Default Setup Type configured when creating or editing a reservation, received the "Attendance must be greater than zero" pop-up. This pop-up appeared at the top of the Attendance & Setup Type dialog box. This was occurring because the defaults for the process template were being validated prior to allowing the users to input the number of attendees and the setup type.

Fix: Modified functionality so that the "Attendance must be greater than zero" pop-up is only displayed if users do not enter an attendance value and submit.

Internal Reference # 33758
(For EMS Internal Use Only)

Everyday User Could Create a Booking with an Invalid Email Address Format on the 1st Contact

For web process templates with the “Require Contact Email” parameter enabled, if users entered an invalid email address format in the 1st Contact Email Address field (e.g., esmith instead of esmith@gmail.com), the reservation would be created without an error message explaining the email address format was invalid.

Fix: Modified functionality so that an error message appears when users attempt to create a reservation with an invalid email address format in the 1st Contact Email Address field.

Internal Reference # 35060
(For EMS Internal Use Only)

Unable to Create Reservations When Cost Center Number Contains "&" Character

When a Cost Center Number containing an "&" character was added in EMS Desktop Client, the EMS Web App was unable to create reservations associated with that Cost Center Number. EMS Web App users would receive an error stating: "Cost Center is required and must be entered." 

Fix: Modified functionality so that Cost Center Numbers containing "&" are accepted in EMS Web App and reservations associated with those Cost Center Numbers can be made.

Internal Reference # 35112
(For EMS Internal Use Only)

Latency Experienced When Pulling Room List

EMS for Outlook and EMS Web App users experienced latency when creating a reservation and searching for a room.

Fix: Modified functionality so that users do not experience any latency when searching for a room.

Internal Reference # 39205
(For EMS Internal Use Only)

Edit Reservation
Room Request

 

Billing Reference and PO Numbers Did Not Update When Group Was Changed

When creating a reservation, if users selected a group with an associated billing reference or PO number (from the Reservation Details tab) and then changed their selection to a different group, and the parameter "Update Billing Information When Changing Group" is set to "YES," the billing reference or PO number would not update with the new group selection.

Fix: Modified functionality so that when users select a different group and the "Update Billing Information When Changing Group" is enabled, the billing reference or PO number updates and displays the correct number for the group.

Internal Reference # 36709
(For EMS Internal Use Only)

Unable to Create Reservations with Billing Reference Numbers Containing All Zeros

EMS Web App users were unable to create a reservation if they selected a billing reference number that contained all zeroes (from the Reservation Details tab). They received the error message: "Billing Reference is required and must be valid."

Fix: Modified functionality so that users can successfully create reservations with billing reference numbers that contain all zeroes.

Internal Reference # 38391
(For EMS Internal Use Only)

Edit Reservation

Unable to Edit Date/Time When Resolving Room Conflict (Calendar-Enabled Templates Only)

When editing a booking on calendar-enabled templates, the date and time filters were locked (uneditable) on the Edit Booking screen, and users were unable to search for a new room and resolve room conflicts.

Fix: Modified functionality so that the date and time filters are unlocked and users can search for an alternate room and resolve room conflicts.
