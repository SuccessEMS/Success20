Create or Edit Reservations

EMS for Outlook Add-in Checks for Connection to EMS Platform Services When Offline

While in an offline state, EMS for Outlook will continue to look for a connection to EMS Platform Services and automatically reconnect if a connection becomes available. Previously if a connection became available, the user had to restart Outlook in order to use EMS for Outlook.

If EMS for Outlook cannot connect, the user will now receive the following message:

"EMS for Outlook connection failed. Do not make any changes to meetings with room bookings while disconnected. EMS for Outlook will continue to look for a connection."

Once EMS for Outlook detects a connection, users can create and modify reservations in the add-in.

Internal Reference # 39372
(For EMS Internal Use Only)

Create or Edit Reservation

Reservations Incorrectly Updated When Rooms Were Changed to Rooms in Building from a Different Time Zone

For both single and recurring reservations, when users changed a room to one in a building from a different time zone, the reservation updated incorrectly. For example, in single occurrence reservations, only one room displayed in the Selected Rooms tab although more than one room was selected, and in recurring reservations, the Days Available column displayed incorrect values.

Fix: Modified functionality so that when users change a room to one in a building from a different time zone, the reservation updates correctly.

Internal Reference # 35985
(For EMS Internal Use Only)

Error Message "You cannot book this Room after..." Displayed the Incorrect Date

In the EMS for Outlook add-in, the error message "You cannot book this Location after..." displayed the date 1/1/0001 instead of the date specified in the Booking Rules tab of Everyday User process template dialog of the EMS Desktop Client.

Fix: Modified functionality so that the error message "You cannot book this Location after..." displays the date specified in the Booking Rules tab of Everyday User process template dialog of the EMS Desktop Client.

Internal Reference # 37128
(For EMS Internal Use Only)

Shared Calendar Users Were Unable to Select Second Room in Recurring Reservations

Some shared calendar users were unable to select a second room in a recurring reservation. For example, a user tried to make a recurring reservation where a room was available three out of four days. The user added the room for three of the four days; however, when the user tried to select a second room for the fourth day, the user was unable to select the room.

Fix: Modified functionality so that users can select a second room in a recurring reservation.
