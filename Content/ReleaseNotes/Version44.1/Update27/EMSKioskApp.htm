Locate a Person

 

EMS Kiosk App "Locate a Person" Results Prompt Was Misleading

When using the Locate a Person search option on the landing page to find reservations, the "your name" text in the results prompt was misleading:

"Here are your results for '[name]'. Choose your name to continue."

The "your name" text was confusing because users may not be searching for their own reservations. They may be searching for another person's reservations or for a group's reservations.

Fix: Modified the Locate a Person search results prompt so that the text is not misleading. The new prompt now states:

"Here are your results for '[name]'. Please select a User/Group to view reservations on behalf of.'"

Internal Reference # 34570
(For EMS Internal Use Only)

User Sessions

 

User Sessions Terminated Without Requiring Timeout/Logoff When Kiosk Profile Parameter Was Set to "Yes"

When the parameter "Require Timeout/Logoff to Terminate User Session" was set to "Yes" in Kiosk Profiles (Legacy version or older), the EMS Kiosk App was ending user sessions when users went back to the main menu instead of ending the sessions only when users logged off or their sessions timed out.

Fix: Modified functionality so that when the parameter "Require Timeout/Logoff to Terminate User Session" is set to "Yes," the EMS Kiosk App does not end user sessions unless users log off or their sessions time out.

Internal Reference # 36619
(For EMS Internal Use Only)

Provisioning

 

EMS Kiosk App Lost Provisioning After Session Expiration

After the EMS Kiosk App session expired, the Kiosk needed to be provisioned.

Fix: Modified functionality so that the EMS Kiosk App provisioning survives token expiration.

Internal Reference # 37125
(For EMS Internal Use Only)
