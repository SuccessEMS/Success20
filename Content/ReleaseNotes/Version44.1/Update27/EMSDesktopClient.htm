Fixes

Area

Description

Web Deploy

Users Could Not Log In When Using Windows Authentication and Connecting with Web Deploy Service

When using Windows Authentication and connecting using the Web Deploy Service, users could not log in.

Fix: Modified functionality so that users can log in when using Windows Authentication and connecting using the Web Deploy Service.

Internal Reference # 37999
(For EMS Internal Use Only)

Confirmation Settings

Date Format Error When Specifying Confirmation Settings

When specifying confirmation settings, if users entered the date in the format dd/mm/yyyy instead of mm/dd/yyyy, they would receive the error message “Argument ‘Expression’ is not a valid value.” Also, this issue sometimes prevented users from sending confirmations.

Fix: Modified functionality so that when specifying confirmation settings, the date format dd/mm/yyyy is valid, and users can enter the date in this format without receiving an error message.

Internal Reference # 38836
(For EMS Internal Use Only)

ActiveReports
Client Web

ActiveReports Library Not Licensed

When users generated a confirmation or report, a message apppeared at the bottom of the confirmation/report stating that the document was created using an evaluation version of ActiveReports and that only a licensed user may legally create reports for use in production.

Fix: Modified functionality so that the ActiveReports license is valid, and the invalid license message does not appear.

Internal Reference # 38828
(For EMS Internal Use Only)
