Group Notifications

Group Notifications Prevented for All Buildings on Rule When One Building Had a Special Date Configured

For a group notification rule configured with two or more buildings, the special dates on any of the configured buildings were preventing notifications from going out for the non-special date buildings on the notification that day.

For example, both Building A and Building B share a group notification rule. Building A has July 4 configured as a special date, but Building B does not, and Building B has bookings on July 4. Even though Building B's bookings should trigger a group notification, the notification is not sent because of the special date configured for Building A.

Fix: Modified functionality so that only buildings with special dates are prevented from triggering group notifications.

Internal Reference # 27922
(For EMS Internal Use Only)

Reservation Wizard

Unable to Select LDAP-Assigned Process Templates in the Reservation Wizard

EMS Desktop Client users were unable to select LDAP-assigned process templates in the Everyday User Applications tab of Reservation Wizard when assigning an everyday user to a reservation.

Fix: Modified functionality so that EMS Desktop Client users are able to select LDAP-assigned process templates in the Everyday User Applications tab of Reservation Wizard.

Internal Reference # 37476
(For EMS Internal Use Only)

Custom Reports

Custom Reports Not Working in Some EMS Desktop Client Versions

Custom reports was not working in EMS Desktop Client version 23.1 or higher.

Fix: Restored functionality so that custom reports work on all Desktop Client versions.

Internal Reference # 38711
(For EMS Internal Use Only)

Web Deploy

Users Could Not Log In When Using Windows Authentication and Connecting with Web Deploy Service

When using Windows Authentication and connecting using the Web Deploy Service, users could not log in.

Fix: Modified functionality so that users can log in when using Windows Authentication and connecting using the Web Deploy Service.

Internal Reference # 37999
(For EMS Internal Use Only)

Confirmations & Invoices

Confirmations and Invoices Displayed Incorrect Information

For a reservation with multiple bookings, the confirmation and invoice displayed all resources and pricing under the last booking date.

Fix: Modified functionality so that invoices and confirmations display each booking under its correct date.

Internal Reference # 39933
(For EMS Internal Use Only)
