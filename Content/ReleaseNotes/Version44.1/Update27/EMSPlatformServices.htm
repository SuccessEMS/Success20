Logging

"Always On" Detailed Logging for Error Requests

When an error occurs during a request, the complete set of log entries for the request are now captured and outputted as follows:

  • On a successful request, the platform will only output logs at the default or specified log level (e.g. "Warning").
  • On an erroneous request, the platform will output fine-grained logs for that request (e.g. all DEBUG/INFO log entries).

Skype for Business > AutoDiscover URLs

EMS Skype for Business Not Working with Internal and External AutoDiscover URLs

For customers with internal and external AutoDiscover URLs, EMS Web App users were unable to add reservations to their Skype accounts. The reservations were created in EMS Web App, but users received the error message: "Your reservation was created, but there was a problem creating the meeting in your conferencing system."

Fix: Modified functionality so that users no longer receive an error and the reservation is added to their Skype account.
