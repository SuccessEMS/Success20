Check In

Check In Button and Automatic Cancellation Were Being Activated Incorrectly

The Check In button was being activated based on the setup or reserved time instead of the event time. Additionally, automatic cancellation was being applied inconsistently (e.g., applying automatic cancellation based on the setup time instead of the interval after check in specified in EMS Desktop Client settings).

Fix: Modified functionality so that the Check In button is activated based on the event time and automatic cancellation cancels the room based on the interval specified in EMS Desktop Client settings.

Internal Reference # 39543
(For EMS Internal Use Only)

Badge Reader

Unable to Scan Badges that Have Badge Numbers Longer Than Nine Digits

If a badge number longer than nine digits was entered in the Badge Number field (EMS Desktop Client > Groups > Edit Group > Other tab ), users would receive an error message when they scanned their badges stating that the EMS Room Sign App was unable to find the users.

Fix: Modified functionality so that users no longer receive an error when they scan their badges.

Internal Reference # 38821
(For EMS Internal Use Only)

ArmorActive

 

ArmorActive Enclosure Was Not in Sync with EMS Room Sign App

The ArmorActive LED light was not in sync with the EMS Room Sign App status after the device polling interval time passed. The EMS Room Sign App was not triggering a sound when the device polling interval time passed.

Fix: Modified functionality so that the EMS Room Sign App triggers a sound when the device polling interval time passes. This results in the enclosure LED light syncing with the EMS Room Sign App status.

Internal Reference # 36960
(For EMS Internal Use Only)
