(For EMS Internal Use Only)

Installation and Permissions

 

EMS.Glance.dll moved to EMS Desktop Client installer and EMS Room Sign App configuration permission added to user processes

Previously, the permission to configure EMS Room Sign App was controlled by the EMS.Glance.dll, which had to be manually installed on the computer of each user who needed permission. As of Update 32, the EMS.Glance.dll has been moved to the standard EMS Desktop Client installer package, and the permission to configure EMS Room Sign App is now granted by admins.

Admins can apply the EMS Room Sign App permission to users and user process templates in the EMS Desktop Client (processes section, Special > Allow EMS Room Sign Administration):

  • With EMS Room Sign administration permission, users can access the EMS Glance Settings menu button in EMS Desktop Client and all the EMS Room Sign App settings accessed through the button. Additionally, users can now access these EMS Room Sign App settings through the Configuration menu: 
    • Configuration > Room Sign > Deregister Device
    • Configuration > Room Sign > Global Settings
    • Configuration > Room Sign > Profiles
    • Configuration > Room Sign > Refresh All Units
  • Without EMS Room Sign administration permission, users do not see the EMS Glance Settings menu button and cannot access the EMS Room Sign App configuration settings through the menu button or the Configuration menu.

Software Updates

Older EMS Database Versions were not auditing the latest ClientConnection in tblServerVersion table

When older EMS Desktop Client versions connected to the database, they were not receiving the alert that a newer version was available. The ClientVersion column in tblServerVersion was being updated with the last version of the Desktop Client used instead of the latest version.

Fix: Modified functionality so that when older Desktop Client versions connect to the database, they receive the alert that a newer version is available.

EMS for Outlook License Checkbox

For users with an EMS for Google Calendar license, an 'Enable for EMS for Google Calendar' checkbox should be available. After checking these boxes, the tab, EMS for Google Calendar Conflicts will appear.

Internal Reference # 42989
(For EMS Internal Use Only)

Audit Tab

Editing Critical Room Information Was Recorded in the Audit Tab As a Change to the Everyday User Process Template

For rooms tied to an Everyday User Process Template, when a user modified critical room information, such as the room code, the user was incorrectly recorded on the Audit tab as changing the Everyday User Process Template.

Fix: Modified functionality so that the Audit tab for Everyday User Process Templates only records changes made to Everyday User Process Templates.

Internal Reference # 40153
(For EMS Internal Use Only)

EMS Email Services

 

Windows Service Not Sending Email

EMS Email Services (including no-show, notifications, and auto reports) were not sending emails to end users or admins.

Fix: Modified functionality so that email is successfully handed off to the SMTP server.

Internal Reference # 42195
(For EMS Internal Use Only)

Modify Booking

Modify Booking Details Security Process Not Allowing Create New Category on Bookings

Modify Booking Details (expert user process template) was not allowing the user to add, edit, or delete the category and resources on any booking if the user also had access to that category, if the user had at least view-only access to the booking room, or if the user had access to the status the booking was in.

Fix: Modified functionality so that the Modify Booking Details security process allows users to add, edit, or delete the category and resources.
