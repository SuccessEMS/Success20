Internal Reference # 42911
(For EMS Internal Use Only)

Installation and Permissions

 

EMS.Glance.dll moved to EMS Desktop Client installer and EMS Room Sign App configuration permission added to user processes

Previously, the permission to configure EMS Room Sign App was controlled by the EMS.Glance.dll, which had to be manually installed on the computer of each user who needed permission. As of Update 32, the EMS.Glance.dll has been moved to the standard EMS Desktop Client installer package, and the permission to configure EMS Room Sign App is now granted by admins.

Admins can apply the permission for EMS Room Sign App configuration to users and user process templates in the EMS Desktop Client (processes section, Special > Allow EMS Room Sign Administration):

  • With EMS Room Sign configuration permission, users can access the EMS Glance Settings menu button in EMS Desktop Client and all the EMS Room Sign App settings accessed through the button. Additionally, users can now access these EMS Room Sign App settings through the Configuration menu: 
    • Configuration > Room Sign > Deregister Device
    • Configuration > Room Sign > Global Settings
    • Configuration > Room Sign > Profiles
    • Configuration > Room Sign > Refresh All Units
  • Without EMS Room Sign configuration permission, users do not see the EMS Glance Settings menu button and cannot access the EMS Room Sign App configuration settings through the menu button or the Configuration menu.

Software Updates

Older EMS Database Versions were not auditing the latest ClientConnection in tblServerVersion table

When older EMS Desktop Client versions connected to the database, they were not receiving the alert that a newer version was available. The ClientVersion column in tblServerVersion was being updated with the last version of the Desktop Client used instead of the latest version.

Fix: Modified functionality so that when older Desktop Client versions connect to the database, they receive the alert that a newer version is available.
