Fixes

Area

Description

Email Notification Service

 

Email Notification Service Did Not Function After Receiving Error

When a mail server returned a 554 error code, the Email Notification Service stopped sending out any other emails.

Fix: Modified functionality so that email is sent based on notification service settings.

Internal Reference # 41126
(For EMS Internal Use Only)

Room Utilization Report

 

Room Utilization Report Not Functioning When Utilizing Special Dates

Room Utilization Report data with Special dates were incorrect. All options pertaining to 'Availability Based On' returned the same incorrect data.

Fix: Modified functionality so that Room Utilization Reports include correct data.

Internal Reference # 37324
(For EMS Internal Use Only)

EMS for Google Calendar

 

EMS for Outlook License Checkbox

For users with an EMS for Google Calendar license, an 'Enable for EMS for Google Calendar' checkbox should be available. After checking these boxes, the tab, EMS for Google Calendar Conflicts will appear.

Fix: Modified functionality so that the appropriate checkboxes and tabs appear for EMS for Google Calendar license holders.

Internal Reference # 42989
(For EMS Internal Use Only)

Audit Tab

Editing Critical Room Information Was Recorded in the Audit Tab As a Change to the Everyday User Process Template

For rooms tied to an Everyday User Process Template, when a user modified critical room information, such as the room code, the user was incorrectly recorded on the Audit tab as changing the Everyday User Process Template.

Fix: Modified functionality so that the Audit tab for Everyday User Process Templates only records changes made to Everyday User Process Templates.

Internal Reference # 40153
(For EMS Internal Use Only)

EMS Email Services

 

Windows Service Not Sending Email

EMS Email Services (including no-show, notifications, and auto reports) were not sending emails to end users or admins.

Fix: Modified functionality so that email is successfully handed off to the SMTP server.

Internal Reference # 42195
(For EMS Internal Use Only)

Modify Booking

Modify Booking Details Security Process Not Allowing Create New Category on Bookings

Modify Booking Details (expert user process template) was not allowing the user to add, edit, or delete the category and resources on any booking if the user also had access to that category, if the user had at least view-only access to the booking room, or if the user had access to the status the booking was in.

Fix: Modified functionality so that the Modify Booking Details security process allows users to add, edit, or delete the category and resources.

Internal Reference # 43009
(For EMS Internal Use Only)
