Parameter

Change to Parameter SAML_SignAuthnRequests

There has been a change to the parameter, SAML_SignAuthnRequests, which controls whether or not SAML requests go to the IdP. This can now be controlled and modified from a toggle in the Admin UI, which reflects the parameter's current state.

EMS Internal Reference # 38968
(For EMS Internal Use Only)

Users can now get states of the categories while adding a service order to Booking.

(For EMS Internal Use Only)

Fixes

Area

Description

API

Booking Information Changes Missing in History Tab

When critical booking changes were made, the booking history was not being updated through the EMS Platform Services API.

Fix: Modified functionality so that changes to booking's critical information (e.g., date, time, room, attendee, etc.) are captured in the booking's history tab in the EMS Desktop Client.

EMS Internal Reference # 42850
(For EMS Internal Use Only)

 

EMS Web App Was Not Connecting to EMS Platform

Due to SAML configuration issues, the EMS Web App was not connecting to the EMS Platform.

Fix: Resolved SAML configuration issues so that the EMS Web App can connect to the EMS Platform.

EMS Internal Reference # 42212
(For EMS Internal Use Only)
 

Group Email Address Incorrect When Client Token Used

When making the POST /bookings/actions/search REST call, the same email address in the Group model was listed under all Groups that were returned with reservations (if a Client Token was used for the x-ems-api token).

Fix: Modified functionality so that the correct Group Email Address is returned, as an /authentication token returns the Group model data.

EMS Internal Reference # 40001
(For EMS Internal Use Only)
 

Reservation Not Inserted When AltTempContact Parameter Exceeded 50 Characters

When a value for altTempContact exceeded 50 characters, a reservation appeared to be created but was in fact an orphaned booking.

Fix: Modified functionality so that the parameter field value is validated prior to insertion and an error is returned if the values do not meet the requirements.

EMS Internal Reference # 41134
(For EMS Internal Use Only)

SAML Authentication

 

Signature Validation Failing (SAML)

Some customers' SAML configuration was not allowing SAML Responses to validate in the EMS Platform (using a Shibboleth IdP).

Fix: Modified functionality so that signatures can be validated.

EMS Internal Reference # 41651
(For EMS Internal Use Only)
