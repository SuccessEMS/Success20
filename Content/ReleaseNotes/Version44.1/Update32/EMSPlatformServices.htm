Enhancements

Area

Description

Parameters

Added parameter Events2HVAC

Events2HVAC parameter — Get Changed Bookings — has been added so that the integrator can get bookings that changed within a timeframe.

The following new query fields have been added to the booking search (filter on tblBooking.DateChanged):

  • minDateChanged - filter tblBooking.DateChanged >= value
  • maxDateChanged - filter tblBooking.DateChanged <= value

EMS Internal Reference # 29333
(For EMS Internal Use Only)

Authentication

ADSF/SAML migration to Azure

SAML/ADSF authentications now work using an Azure IDP.

EMS Internal Reference # 42145
(For EMS Internal Use Only)

Service Orders

State ID can be set when adding or editing a service order

Users can now set the state ID while adding a service order to a booking or while editing a service order in a booking.

The following routes have been updated to include state Id of service in the request body:

  • POST/bookings/{id}/serviceorders
  • PATCH/serviceorders/{id}

EMS Internal Reference # 42836 and 42838
(For EMS Internal Use Only)

Service Orders

Added new GET Category States route

A new route, GET/Categories/{id}/states, has been added. Users can now get states of the categories while adding a service order to booking.

  • Request parameter: Category Id { Required}
  • Response: Return state id and description and categoryID

EMS Internal Reference # 42930
(For EMS Internal Use Only)

Parameters

Change to parameter SAML_SignAuthnRequests

There has been a change to the parameter, SAML_SignAuthnRequests, which controls whether SAML requests go to the IdP. This can now be controlled and modified from a toggle in the Admin UI, which reflects the parameter's current state.

EMS Internal Reference # 38968
(For EMS Internal Use Only)

EMS Kiosk App

Added paging to GET - /kiosk/{id}/diagrams endpoint

To implement lazy loading for room diagrams in EMS Kiosk App, paging has been added to the GET /kiosk/{id}/diagrams endpoint as follows: 

  • added page Query String parameter with default of 1
  • added pageSize Query String parameter with default of 5
  • added sortBuildingId parameter because EMS Kiosk App needs to have diagrams sorted by a default building ID and then have the remaining results sorted alphabetically by building name

EMS Internal Reference # 43048
(For EMS Internal Use Only)

Booking Search Call

Added 'HideGroup' room property to booking search call

In the booking search call (/bookings/actions/search), a 'hideGroup' property has been added to the BookingRoomModel being returned. The 'hideGroup' property is used to determine when to replace a group name with the Private label when viewing room details in EMS Web App.

EMS Internal Reference # 43067
(For EMS Internal Use Only)

Skype for Business

Skype for Business Meeting Option Parameter Added

A new meeting option parameter has been added to the EMS Platform Services Admin UI under Skype for Business on the Conferencing tab. The parameter These people don't have to wait in the lobby is a drop-down field with the following values:

  • Only me, the meeting organizer
  • People I invite from my company
  • Anyone from my organization
  • Anyone (no restrictions

The default value is Anyone from my organization.

EMS Internal Reference # 42219 and 43684
(For EMS Internal Use Only)

Fixes

Area

Description

API Calls

Group email address incorrect when client token was used

When making the POST /bookings/actions/search REST call, the same email address in the Group model was listed under all Groups that were returned with reservations (if a Client Token was used for the x-ems-api token).

Fix: Modified functionality so that the correct Group Email Address is returned, as an /authentication token returns the Group model data.

EMS Internal Reference # 40001
(For EMS Internal Use Only)

Parameters

Reservation not inserted when AltTempContact parameter exceeded 50 characters

When a value for altTempContact exceeded 50 characters, a reservation appeared to be created but was in fact an orphaned booking.

Fix: Modified functionality so that the parameter field value is validated prior to insertion and an error is returned if the values do not meet the requirements.

EMS Internal Reference # 41134
(For EMS Internal Use Only)

SAML Authentication

 

Signature validation failing (SAML)

Some customers' SAML configurations were not allowing SAML Responses to validate in the EMS Platform (using a Shibboleth IdP).

Fix: Modified functionality so that signatures can be validated.

EMS Internal Reference # 41651
(For EMS Internal Use Only)

API

 

Request Body was missing for Get UDF Values for Multiple Rooms

The Get UDF Values for Multiple Rooms call did not include the Request Body section with the RoomIds array variable. The RoomIds array is a required field in the body for the Get UDF Values for Multiple Rooms call.

Fix: Added the Request Body section to the Get UDF Values for Multiple Rooms call.

EMS Internal Reference # 43077
(For EMS Internal Use Only)

API

 

Deleted Remove Delegates Response

The Remove Delegates call requires a response in an array; however, when calling Remove Delegates, the response received was a single message not in an array. This issue was causing some customers' code to fail.

Fix: Modified functionality so that the Remove Delegates call no longer requires a response in an array

EMS Internal Reference # 43115
