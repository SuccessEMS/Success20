LDAP Authentication > Login Page

EMS Kiosk App now applies customized label to the Name field on the user login page

When using LDAP as the authentication method, the EMS Kiosk App now respects the label entered into Settings > Parameters > Everyday User Applications > Label for Login box (through EMS Desktop Client) and applies this label to the Name field on the login page of the EMS Kiosk App. The login page appears when users are not logged in, and they try to access the My Reservations page or Make a Reservation page.

Internal Reference # 39382
(For EMS Internal Use Only)

Make a Reservation

Room search feature filter added to Make a Reservation page

A feature filter has been added to the Make a Reservation page. Users can now select the Feature filter icon and select one or more room features (such as single monitor and dual monitor) to narrow the room search results. Note that the feature filter will only display if the EMS Kiosk profile has a room count less than 30.

Feature Filter Icon

Internal Reference # 41606, 42200
(For EMS Internal Use Only)

Kiosk Labels

EMS Kiosk App respects all label values defined in EMS Desktop Client

The EMS Kiosk App now respects all labels defined in the Kiosk parameters tab of EMS Desktop Client (System Administration > Settings > Parameters > Kiosk). The following labels defined in the Kiosk parameters tab will be honored and displayed in the EMS Kiosk App: 

Today's Events area:

  • Room Type (name of filter and top of filter results)
  • Building (filter by floor—top of filter results)

Make a Reservation area:

  • Room Type (name of filter and top of filter results)
  • When Hovering over a pin—Room and Room Type labels

Floor Map area:

  • When Hovering over a pin—Room and Room Type labels

Locate a Group area:

  • Menu Name (Group)
  • Results (Room Type and Room Column Labels)

Internal Reference # 41715
(For EMS Internal Use Only)

Provisioning Page > Authentication

Users must have administrator privileges to access the Provisioning page

For added security, users must be logged in as an administrator to access the EMS Kiosk App Provisioning page. If users are not logged in and try to access the Provisioning page, the message "Re-authentication is required to access Kiosk Provisioning screen" will appear and users will need to enter their admin credentials. If users do not have admin credentials, they can select the Cancel button to be returned to the default page.

Internal Reference # 42166
(For EMS Internal Use Only)

Locate a Person

Updated messaging on Locate a Person

To prevent confusion, the message that displays after users select a person on the Locate a Person page has been updated. The message now reads Locate "xPerson" instead of Continue as "xPerson".

Internal Reference # 42215
(For EMS Internal Use Only)

Make a Reservation and Locate Space

Added lazy loading for room diagrams

To help profiles load more efficiently, lazy loading has been implemented for room diagrams in the sidebar (filters) of the Make a Reservation page and Locate Space page. Lazy loading has been implemented as follows:

  • On page load, only the first group of images loads (4 images per group).
  • When users scroll to the bottom of an image group, an activity indicator will display while another group of images is gathered. After the new group of images has been retrieved, the activity indicator disappears, and the new group is appended to the bottom of the previous group of images. This process repeats until users stop scrolling or until no more image results are available.

Internal Reference # 43011
(For EMS Internal Use Only)
