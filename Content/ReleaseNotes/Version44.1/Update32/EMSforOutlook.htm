Notifications and Conflict Resolution

 

Reservations on rooms that require approval that had setup times specified resulted in a conflict resolution loop

EMS Outlook add-in users were experiencing a conflict resolution loop (users could not exit conflict resolution) for reservations made in the EMS Web App on rooms that require approval (as opposed to self-serve) and that had setup times specified. Users were experiencing the conflict resolution loop when opening the reservations or when trying to dismiss reminders for the reservations.

For example, in the EMS Web App, a room that requires approval is booked for an event with a 30-minute setup time specified. Later, an EMS Outlook add-in user opens the Outlook application 15 minutes before the event starts (the setup time has already begun). Outlook displays a reminder notification for the event, and the user tries to dismiss the notification. However, instead of dismissing the notification, the user is prompted for conflict resolution, and the user cannot exit conflict resolution.

Fix: Modified functionality so that EMS Outlook add-in users are not prompted for conflict resolution when they open reservations on rooms that require approval and/or when they try to dismiss reminders for these types of reservations.

Internal Reference # 43267
