Area

Description

LED

LED Sync Error

Users experienced an LED sync error on EMS Room Sign devices when check-in was present at the same time that 'reserve' was also available. When the user created a reservation in front of an existing meeting that was already allowing check-in, the device displayed the check-in color while there was a reserved meeting in progress. Additionally, the 'check-in' button was still displayed for that future event.

Fix: Modified functionality so that the device should switch to the reserved LED color until the meeting ends and then switch back to the check-in LED status color while hiding the check-in button for the future event.

Internal Reference # 35825
(For EMS Internal Use Only)

Create a Reservation

Error Message Experienced When Creating Future Meetings

When creating a meeting in the future, users were able to create the reservation but experienced a "Room is not available," error message.

Fix: Modified functionality so that the "Room is booked successfully," message is shown when creating a reservation in the future.

Internal Reference # 36607
(For EMS Internal Use Only)

Secondary Authentication

Secondary Authentication Field Not Clearing

When users entered incorrect values in the EMS Room Sign App, the secondary authentication fields were not clearing.

Fix: Modified functionality so that fields can be cleared when incorrect values are entered into the EMS Room Sign App.
