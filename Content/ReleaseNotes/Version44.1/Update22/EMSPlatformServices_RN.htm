User-Defined Fields (UDFs)

Search Reservations by UDFs

Users can now search reservations by user-defined fields (UDFs). GET/reservation allows searching by one UDF, and POST/reservation allows searching by an array of UDFs.

When filtering by UDF, both key and value filters are required.

  • UDF name filter will search room UDFs (tblUserDefinedField) and join up to tblUDFDefinition to get the UDF name value
    • case insensitive match
  • UDF value filter will search room UDF values
    • exact match, case insensitive
    • list-type UDFs require going to tblUDFDefinitionItem to get the actual configured value string

Event Type IDs

Search Reservations by Event Type IDs

Users can now search reservations by multiple event type IDs and display reservations of filtered event types on digital signs and other displays.

The following have been added to enable searching reservations by multiple event type IDs:

  • An array of eventTypeIds has been added as a search criteria to POST/reservations/action/search
  • The eventTypeId has been added as a search criteria to GET/reservations

(For EMS Internal Use Only)

Web User ID

Search Reservations by Web User ID

Users can now search reservations by web user ID and display reservations of the filtered users on digital signs and other displays.

The webUserID filter has been added to the following routes:

  • GET/reservations (query string parameter)
  • POST/reservatoins/actions/search (part of the request object)

EMS for Outlook

When Authenticating First Time Users Through EMS for Outlook, Security Template Was Not Being Assigned

When authenticating users for the first time through EMS for Outlook, a security template was not being assigned, and the user was receiving an "invalid url" message when logging in to the Outlook add-in.

Fix: Modified functionality so that when authenticating users for the first time in EMS Outlook, the security template configured for the user in the EMS Desktop Client is assigned to the user, and the user is able to log in to the EMS for Outlook add-in.

Internal Reference # 35951
(For EMS Internal Use Only)

API Calls

GET/Bookings Returned Incorrect Combo Room Details

GET/Bookings route was returning component room bookings even though "includeComponentRooms" flag was set to false.

Fix: Modified functionality so that the correct room details (BookingRoomModel) are returned for combo and component rooms, and component room bookings are not returned when the filter "includeComponentRooms" is set to false for the following: 

  • GET /bookings
  • POST /bookings/actions/search
  • /bookings/actions/search/users
