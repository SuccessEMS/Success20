Copying Bookings or Reservations Did Not Update "OriginalStartTimeUTC" on Bookings Table, Causing Platform Services Errors

When copying bookings or reservations, the "OrginalStartTimeUTC" column in the newly created booking was set to null. This resulted in Platform Services erroring and endpoints using Platform Services to error when rendering bookings (for instance, EMS Web App and custom integration points).

Fix: Modified functionality so that "OrginalStartTimeUTC" successfully populates and Platform Services and corresponding access points will operate successfully.
