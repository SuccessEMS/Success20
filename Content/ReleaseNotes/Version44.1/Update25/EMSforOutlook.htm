Create or Edit Reservation

Reservations Incorrectly Updated When Rooms Were Changed to Rooms in Building from a Different Time Zone

For both single and recurring reservations, when users changed a room to one in a building from a different time zone, the reservation updated incorrectly. For example, in single occurrence reservations, only one room displayed in the Selected Rooms tab although more than one room was selected, and in recurring reservations, the Days Available column displayed incorrect values.

Fix: Modified functionality so that when users change a room to one in a building from a different time zone, the reservation updates correctly.

Internal Reference # 35985
(For EMS Internal Use Only)

Error After Creating a Reservation in the EMS for Outlook Add-in

For some EMS for Outlook add-in users, when they used a template to create a reservation, an error occurred after clicking Save & Close or Send, and the reservation was not saved in the EMS database. However, the invite was sent to attendees.

Fix: Modified functionality so that when users create a reservation in the EMS for Outlook add-in, the template information populates appropriately, and the reservation is created with no errors.

Internal Reference # 36800
(For EMS Internal Use Only)

Creating or Updating Reservations through EMS for Outlook Caused Microsoft Outlook to Crash

For some Microsoft Outlook 2010 users, creating or updating reservations through EMS for Outlook caused Microsoft Outlook to crash.

Fix: Modified functionality so that Microsoft Outlook does not crash when users create or update reservations through EMS for Outlook.

Internal Reference # 37689
(For EMS Internal Use Only)

Meeting Time Changes Made Via Drag-and-Drop in Microsoft Outlook Calendar Were Not Updating in EMS Desktop Client

When users modified an EMS for Outlook meeting time via drag-and-drop in the Outlook calendar, the meeting time did not update in EMS Desktop Client.

Fix: Modified functionality so that when users modify an EMS for Outlook meeting time via drag-and-drop in the Outlook calendar, the meeting time will update in EMS Desktop Client.

Internal Reference # 38760
(For EMS Internal Use Only)

Service Time Not Updated When Updating Booking Time in EMS for Outlook Add-in

For meetings with services created in the EMS for Outlook add-in, when users changed the start and end times in the Microsoft Outlook application, the service time on the Services tab was not updated in the EMS for Outlook add-in.

Fix: Modified functionality so that when users change the meeting start and end times in the Microsoft Outlook application, the service time on the Services tab is updated in the EMS for Outlook add-in.

Internal Reference # 38761
(For EMS Internal Use Only)
 

Event Time Updates Made Via Drag-and-Drop or in Outlook Native Window Were Not Updated in the EMS for Outlook Reserved Room Time Bar and EMS Desktop Client

For some EMS for Outlook reservations, when users quickly updated the event time via drag-and-drop or in the Outlook native window, the reserved room’s green time bar displayed the old time in Schedule View of the EMS for Outlook add-in. Additionally, the event time was not updated in EMS Desktop Client.

Fix: Modified functionality so that when users quickly update a reservation’s event time, the reserved room’s green time bar displays the updated event time, and the event time is also updated in EMS Desktop Client.

Internal Reference # 38773
(For EMS Internal Use Only)

Outlook Functionality

Some Outlook Emails with a Delay Rule Were Not Sent If the EMS for Outlook Add-in Was Installed

For some Microsoft Outlook emails with a delay rule, the emails were not sent if the EMS for Outlook Add-in was installed. Instead of being sent, the emails went to the Outlook Outbox.

Fix: Modified functionality so that Outlook emails with a delay rule are sent to their recipients instead of going to the Outlook Outbox.

Internal Reference # 37927
(For EMS Internal Use Only)

Auto-Summary

EMS for Outlook Reservations Auto-summary Text Was Corrupted

Extra characters and HTML tags were being included in confirmation headers, particularly event name and event type.

Fix: Modified functionality so that random symbols do not appear in Reservation auto-summaries being sent after Reservation Creation using EMS for Outlook.

Internal Reference # 38435
(For EMS Internal Use Only)
