Fixes 

Area

Description

EMS Mobile > Browse Locations

Bug 22372
Bug 22789

Bug 19561

Bug 21922

 

Clicking Add New Button Before Locations List Loads Caused Crash

When Users browsed locations and clicked the Add New button before the list finished loading, the app crashed with the error "unfortunately, EMS has stopped".

Fix: Modified the interface so that the button is not available until the list finishes loading. 

Clicking Room Schedule Caused Crash

When users browsed locations and clicked on a room's schedule on the view icon, the app crashed.

Fix: Modified functionality so that users can now open a room's schedule view and continue with the booking process.

Closest Building Not Automatically Searched

When users browsed for space and had Location services activated, EMS Mobile App did not automatically select the nearest building first. 

Fix: Modified functionality to check for location services before searching for space so users can now see available space in the nearest building first.

Long Room Search Results Caused Slowness

When room searches for a building resulted in long lists, users experienced slow load times.

Fix: Changed the display to only show 100 rooms at a time, and users can now click a new Load More Results button to see more than those listed.

EMS Mobile > My Events

Bug 22171

Bug 22361

Bug 22738

Events Listed for Wrong Dates

When viewing the events calendar grid and clicking on a date, the events in w did not match.

Fix: Modified validations so that users can now click on a date in the grid and only see events for that date in the list below. 

Clicking Now Button Selects Wrong Date

When picking dates and then clicking the Now button on the calendar grid, previously selected dates were still selected.

Fix: Modified display so that users now see no dates selected if they click the Now button.

Details Button Incorrectly Available When Clicking Back During a Search

When users had no events and clicked Search, then Details, and then clicked back, the Details button displayed.

Fix: Modified the display so that the Details button does not display when users have no events.

EMS Mobile > New Reservation

Bug 22741

Favorite Rooms Incorrectly Reservable When Booked

When making a new reservation, favorite rooms incorrectly displayed as available when they were already booked.

Fix: Improved functionality so that booked rooms no longer display as available even if they are favorites.

EMS Mobile > New Reservation > Terms and Conditions

Bug 22439

Remove Terms and Conditions from Template Not Updating on EMS Mobile

When Terms and Conditions were removed from a booking template in EMS Desktop Client, the changes did not show in EMS Mobile and the Terms and Conditions link remained visible (although unclickable) on the reservation. 

Fix: Modified functionality so that users can now remove Terms and Conditions from a template and see those change reflected immediately in EMS Mobile.

Clicking Back Button on Androids Closes App
When Android users clicked the Back button from the following pages, the app closed instead of returning them to the home screen.

  • Create a Reservation

IOs-Only Issues 

EMS Mobile > Browse Locations > Filters > Add New

Bug 22781

Newly Added Filters Not Saved

When adding a new filter from Browse Locations, the new filter was not saved and clicking the Save button could cause the app to crash.

Fix: Modified functionality so that users can now add and save filters without the error.
