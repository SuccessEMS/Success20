Cloud Services Installation and Updating

For EMS customers with a Cloud Services subscription, EMS provisions your EMS Cloud environment. EMS also manages and monitors your application environment. Refer to the following topics for more information on how Cloud Services handles installation and updates in your EMS Cloud environment:

  • Installation
  • Schedule of Updates and Fixes

Installation

For Cloud Customers, EMS is responsible for the provisioning of hardware, operating system, database, licensing, upgrades, server maintenance, internet connectivity, and data backups. Cloud services also handles the security of physical space and network access, as well as anti-virus, data loss prevention, upgrades, updates, and security patches.

Schedule of Updates and Fixes

New EMS releases will be applied to your EMS Cloud environment using the following schedule:

  • One week after a release is Generally Available, the release will be applied to your Test Environment.
  • Two weeks after a release is Generally Available, the release will be applied to your Production environment. Note: This is the standard process for minor releases without many UI changes.
  • Five weeks after a release is Generally Available, the release will be applied to your Production environment. Note: This is the standard process for major releases with significant UI changes.
  • Hot fixes will be applied to both Test and Production environments as soon as they are Generally Available.
